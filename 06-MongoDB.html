🍃 מהו MongoDB?

הגדרה

MongoDB הוא מסד נתונים מסוג document-oriented, קוד פתוח וחוצה פלטפורמות

  • מספק ביצועים גבוהים, זמינות גבוהה וסקלביליות אוטומטית
  • עובד על בסיס Collection ו-Document (JSON)
  • רשומה ב-MongoDB היא document - מבנה נתונים המורכב מזוגות שדה-ערך
  • ערכי שדות יכולים לכלול documents מוטמעים, מערכים ומערכים של documents

מאפיינים ויתרונות

📝 JSON Documents

Documents מיוצגים כאובייקטי JSON

💾 BSON Storage

אחסון כ-BSON (Binary JSON)

🎯 Native Types

תואם לטיפוסי נתונים מקוריים

🔗 Embedded Data

מערכים מוטמעים מפחיתים צורך ב-joins

🔍 Flexible Indexing

יכולת indexing גמישה

🌐 Multi-Language

תמיכה בשפות תכנות רבות

למה להשתמש ב-MongoDB?

  • Schemaless - collection אחד מכיל documents שונים
  • מבנה ברור - המבנה של object בודד ברור וקריא
  • ללא joins מורכבים
  • Deep query-ability - תמיכה בשאילתות דינמיות חזקות
  • Scale-Out קל - MongoDB קל להרחבה
  • Replication & High Availability
  • Auto-Sharding
  • Pluggable Storage Engines

📄 MongoDB Document - פורמט JSON

דוגמת Document

{ "first_name": "Paul", "surname": "Miller", "cell": 447557505611, "city": "London", "location": [45.123, 47.232], "profession": ["banking", "finance", "trader"], "cars": [ { "model": "Bentley", "year": 1973, "value": 100000 }, { "model": "Rolls Royce", "year": 1965, "value": 330000 } ] }

שדה ה-_id

כל פריט ב-MongoDB מכיל שדה _id שמשמש כ-Primary Key

  • ניתן ליצור אותו מפורשות או שיוצר אוטומטית על ידי MongoDB
  • _id ממו-indexed אוטומטית - חיפושים לפי _id משתמשים ב-index
  • ברירת מחדל: שדה _id הוא ObjectID - אחד מטיפוסי BSON

🗂️ מודל הנתונים ב-MongoDB

1️⃣ Database

  • Physical Container עבור collections
  • כל database מקבל קבוצת קבצים משלו במערכת הקבצים
  • שרת MongoDB יחיד בדרך כלל מכיל מספר databases

2️⃣ Collection

  • קבוצה של MongoDB documents
  • שווה ערך ל-Table ב-RDBMS
  • Collection קיים בתוך database בודד
  • Collections לא כופים schema
  • Documents בתוך collection יכולים להכיל שדות שונים
  • בדרך כלל, כל ה-documents ב-collection הם בעלי מטרה דומה

3️⃣ Document

  • סט של זוגות key-value בייצוג בינארי (BSON)
  • Dynamic Schema - documents באותו collection לא חייבים אותם שדות
  • שדה משותף יכול להכיל טיפוסי נתונים שונים
  • קידוד BSON מרחיב את JSON ומוסיף טיפוסים נוספים: INT, LONG, DECIMAL, DATE

השוואת מינוחים: RDBMS ↔ MongoDB

💾 Storage Engines ב-MongoDB

מהם Storage Engines?

  • MongoDB ניתן להרחבה עם יכולות חדשות באמצעות pluggable storage engines
  • ניתן לערבב ולהתאים מספר storage engines בתוך cluster יחיד
  • גמישות זו מספקת גישה פשוטה ואמינה לתמיכה בעומסי עבודה מגוונים

🔷 WiredTiger (Default)

Storage engine ברירת המחדל של MongoDB

  • בקרת concurrency ברמת granular
  • דחיסה מקורית (native compression)
  • הביצועים והיעילות הטובים ביותר עבור רוב האפליקציות
  • Document-Level Concurrency Control
  • תמיכה בדחיסת דיסק

🔒 Encrypted Storage Engine

  • מגן על נתונים רגישים ביותר
  • ללא overhead של ביצועים או ניהול של הצפנת filesystem נפרדת
  • מבוסס על WiredTiger
  • חלק מ-MongoDB Enterprise Edition

⚡ In-Memory Storage Engine

  • מספק latency צפוי ואנליטיקה בזמן אמת
  • עבור אפליקציות הדורשות ביצועים מירביים
  • חלק מ-MongoDB Enterprise Advanced
  • Excellent Predictable Latency (99%)
  • לא תומך ב-datasets גדולים מ-RAM

⚠️ MMAPv1 (Deprecated)

  • מסופק לתאימות לאחור בלבד
  • מוצהר deprecated מגרסה 4.0
  • Collection-Level Concurrency Control (לא document-level)
  • אין תמיכה בדחיסה

🔍 MongoDB Indexes

מהם Indexes?

Indexes תומכים בביצוע יעיל של queries ב-MongoDB

  • בלי indexes, MongoDB חייב לסרוק כל document (collection scan)
  • Indexes מאחסנים חלק קטן מה-dataset בצורה קלה למעבר
  • MongoDB יוצר index ייחודי על שדה _id באופן אוטומטי

סוגי Indexes

1️⃣ Single Field Index

// Create single field index db.collection.createIndex({ field: 1 }) // 1 = ascending db.collection.createIndex({ field: -1 }) // -1 = descending // Example: db.products.createIndex({ price: 1 })

2️⃣ Compound Index

// Create compound index on multiple fields db.collection.createIndex({ field1: 1, field2: -1 }) // Example: db.products.createIndex({ category: 1, price: -1 }) // סדר השדות חשוב! // התומך בשאילתות על: // 1. category // 2. category + price // אבל לא רק price

3️⃣ Multikey Index

// Index on array fields db.collection.createIndex({ arrayField: 1 }) // Example: db.products.createIndex({ tags: 1 }) // MongoDB יוצר index entry נפרד לכל אלמנט במערך

4️⃣ Text Index

// Create text index for text search db.collection.createIndex({ field: "text" }) // Example: db.articles.createIndex({ title: "text", content: "text" }) // Search: db.articles.find({ $text: { $search: "mongodb tutorial" } })

5️⃣ Geospatial Index

// 2d index for flat geometry db.collection.createIndex({ location: "2d" }) // 2dsphere index for spherical geometry db.collection.createIndex({ location: "2dsphere" }) // Example query: db.places.find({ location: { $near: { $geometry: { type: "Point", coordinates: [ -73.9667, 40.78 ] }, $maxDistance: 5000 } } })

ניהול Indexes

// List all indexes db.collection.getIndexes() // Drop index db.collection.dropIndex("index_name") db.collection.dropIndex({ field: 1 }) // Drop all indexes (except _id) db.collection.dropIndexes() // Get index statistics db.collection.stats().indexSizes

Index Properties

// Unique index db.collection.createIndex({ email: 1 }, { unique: true }) // Sparse index (only indexes documents with the field) db.collection.createIndex({ field: 1 }, { sparse: true }) // TTL index (auto-delete after time) db.collection.createIndex( { createdAt: 1 }, { expireAfterSeconds: 3600 } ) // Partial index (with filter) db.collection.createIndex( { status: 1 }, { partialFilterExpression: { rating: { $gt: 5 } } } )

💡 Best Practices:

  • צור indexes עבור queries תכופים
  • השתמש ב-compound indexes לשאילתות מרובות שדות
  • מנטר את ביצועי ה-indexes עם explain()
  • זכור: indexes משפרים read אבל מאטים write

🚀 MongoDB Deployment

סוגי Deployment

1️⃣ Standalone

Instance יחיד של MongoDB

  • פשוט ביותר להתקנה
  • מתאים לפיתוח ובדיקות
  • ❌ אין redundancy או high availability
  • ❌ לא מומלץ לייצור
// Start standalone MongoDB mongod --port 27017 --dbpath /data/db // Connect mongo --host localhost:27017

2️⃣ Replica Set

קבוצה של MongoDB instances ששומרים את אותו dataset

  • ✅ High Availability
  • ✅ Data Redundancy
  • ✅ Automatic Failover
  • Primary node מקבל כל ה-writes
  • Secondary nodes משכפלים את הנתונים
  • מינימום 3 nodes (או 2 + arbiter)

3️⃣ Sharded Cluster

מחלק נתונים על פני מספר machines

  • ✅ Horizontal Scaling
  • ✅ Large Datasets
  • ✅ High Throughput
  • מורכב מ: shards, config servers, mongos routers

🔄 MongoDB Replication

מהי Replication?

Replication מסנכרנת נתונים על פני מספר servers

  • מספקת redundancy וזמינות גבוהה
  • מגנה מפני אובדן server בודד
  • מאפשרת disaster recovery
  • מספקת read capacity נוספת

Replica Set - רכיבים

👑 Primary

מקבל את כל פעולות הכתיבה

📚 Secondary

משכפל מה-Primary

🗳️ Arbiter

משתתף בהצבעות (לא מחזיק data)

הקמת Replica Set

// Initialize replica set rs.initiate({ _id: "myReplicaSet", members: [ { _id: 0, host: "mongodb0.example.net:27017" }, { _id: 1, host: "mongodb1.example.net:27017" }, { _id: 2, host: "mongodb2.example.net:27017" } ] }) // Check status rs.status() // Add member rs.add("mongodb3.example.net:27017") // Remove member rs.remove("mongodb3.example.net:27017")

Automatic Failover

כאשר Primary נכשל:

  • Secondary nodes מזהים את הכשל
  • מתבצעת הצבעה אוטומטית
  • Secondary עם הנתונים העדכניים ביותר נבחר כ-Primary חדש
  • התהליך לוקח בדרך כלל 10-12 שניות

📊 MongoDB Sharding

מהו Sharding?

Sharding מחלק נתונים על פני מספר machines

  • מאפשר horizontal scaling
  • תומך ב-datasets עם תעבורה גבוהה
  • מפחית את העומס על server בודד

רכיבי Sharded Cluster

📦 Shards

מחזיקים את הנתונים (replica sets)

🗺️ Config Servers

מאחסנים metadata של ה-cluster

🔀 Mongos

Query routers - מנתבים queries

Shard Keys

// Enable sharding on database sh.enableSharding("myDatabase") // Shard a collection sh.shardCollection("myDatabase.myCollection", { userId: 1 }) // Range-based sharding sh.shardCollection("myDatabase.logs", { timestamp: 1 }) // Hashed sharding (better distribution) sh.shardCollection("myDatabase.users", { _id: "hashed" }) // Compound shard key sh.shardCollection("myDatabase.events", { country: 1, userId: 1 })

⚠️ בחירת Shard Key חשובה מאוד!

  • לא ניתן לשנות shard key אחרי sharding
  • בחר key עם cardinality גבוה
  • הימנע מ-monotonically increasing keys (כמו timestamp)
  • שקול את דפוסי הגישה לנתונים

💳 MongoDB Transactions

Single-Document Transactions

MongoDB מבטיח atomicity ברמת document בודד

  • כל write operation ל-document בודד הוא atomic
  • Embedded documents ו-arrays תופסים קשרים בתוך document אחד
  • Single-document atomicity מבטל צורך ב-multi-document transactions במקרים רבים

Multi-Document Transactions

החל מגרסה 4.0: תמיכה ב-multi-document transactions

  • עבור מספר operations, collections, databases ו-documents
  • "All-or-Nothing" - הכל מצליח או הכל נכשל
  • כשtransaction מתבצע, כל השינויים נשמרים
  • אם operation נכשל, ה-transaction מתבטל וכל השינויים נמחקים
  • עד שtransaction מתבצע, השינויים לא נראים מחוץ ל-transaction

שימוש ב-Transactions

// Start a session const session = db.getMongo().startSession() // Start transaction session.startTransaction() try { const accountsCollection = session.getDatabase("bank").accounts // Debit from account A accountsCollection.updateOne( { _id: "A" }, { $inc: { balance: -100 } }, { session } ) // Credit to account B accountsCollection.updateOne( { _id: "B" }, { $inc: { balance: 100 } }, { session } ) // Commit transaction session.commitTransaction() print("Transaction committed") } catch (error) { // Abort on error session.abortTransaction() print("Transaction aborted: " + error) } finally { session.endSession() }

Transaction Methods

// Start transaction Session.startTransaction() // Commit transaction Session.commitTransaction() // Abort transaction Session.abortTransaction()

⚠️ Timeouts & Locks

  • ברירת מחדל: transaction מחכה עד 5 אלפיות שניה לקבלת locks
  • אם לא מצליח לקבל locks, ה-transaction מתבטל
  • ניתן להגדיר timeout ארוך יותר בהתאם לצורך

📝 Create (Insert)

🔍 Read (Query)

// Find all documents db.users.find() // Find with pretty print db.users.find().pretty() // Find one db.users.findOne() // Find with filter db.users.find({ age: { $gte: 25 } }) // Find with projection (select specific fields) db.users.find( { age: { $gte: 25 } }, { name: 1, email: 1, _id: 0 } ) // Limit results db.users.find().limit(10) // Skip results (offset) db.users.find().skip(5).limit(10) // Sort results db.users.find().sort({ age: -1, name: 1 }) // Count documents db.users.find({ age: { $gte: 25 } }).count() db.users.countDocuments({ age: { $gte: 25 } })

Query Operators

// Comparison operators db.products.find({ price: { $gt: 100 } }) // greater than db.products.find({ price: { $gte: 100 } }) // greater than or equal db.products.find({ price: { $lt: 1000 } }) // less than db.products.find({ price: { $lte: 1000 } }) // less than or equal db.products.find({ price: { $eq: 500 } }) // equal db.products.find({ price: { $ne: 500 } }) // not equal db.products.find({ price: { $in: [100, 200, 300] } }) db.products.find({ price: { $nin: [100, 200] } }) // Logical operators db.products.find({ $and: [ { price: { $gte: 100 } }, { price: { $lte: 1000 } } ] }) db.products.find({ $or: [ { category: "electronics" }, { category: "computers" } ] }) db.products.find({ price: { $not: { $gt: 1000 } } }) // Element operators db.products.find({ discount: { $exists: true } }) db.products.find({ quantity: { $type: "number" } }) // Array operators db.products.find({ tags: "sale" }) // contains db.products.find({ tags: { $all: ["sale", "featured"] } }) db.products.find({ tags: { $size: 3 } }) // exact array length // Regex db.products.find({ name: /^AC/ }) // starts with AC db.products.find({ name: { $regex: "laptop", $options: "i" } }) // case insensitive

✏️ Update

// Update single document db.users.updateOne( { _id: 1 }, { $set: { age: 31 } } ) // Update multiple documents db.users.updateMany( { age: { $lt: 25 } }, { $set: { status: "young" } } ) // Replace entire document db.users.replaceOne( { _id: 1 }, { name: "New Name", email: "new@example.com", age: 30 } ) // Update operators db.users.updateOne( { _id: 1 }, { $set: { email: "newemail@example.com" }, // set field $inc: { age: 1 }, // increment $mul: { score: 1.1 }, // multiply $rename: { "name": "fullName" }, // rename field $unset: { tempField: "" }, // remove field $min: { lowScore: 50 }, // set if lower $max: { highScore: 100 }, // set if higher $currentDate: { lastModified: true } // set current date } ) // Array update operators db.users.updateOne( { _id: 1 }, { $push: { tags: "new-tag" }, // add to array $pull: { tags: "old-tag" }, // remove from array $addToSet: { tags: "unique-tag" }, // add if not exists $pop: { tags: 1 } // remove last (1) or first (-1) } ) // Upsert (update or insert) db.users.updateOne( { email: "new@example.com" }, { $set: { name: "New User", age: 25 } }, { upsert: true } )

🗑️ Delete

// Delete single document db.users.deleteOne({ _id: 1 }) // Delete multiple documents db.users.deleteMany({ age: { $lt: 18 } }) // Delete all documents in collection db.users.deleteMany({}) // Old method (deprecated) db.users.remove({ _id: 1 }) db.users.remove({ age: { $lt: 18 } })

💪 תרגילים מעשיים

תרגיל 1: התקנה וטעינת נתונים

מטרה: ייבוא נתונים מקובץ JSON ל-MongoDB

  1. בצע התקנה של MongoDB
  2. ייבא את כל הנתונים מקובץ products.json

// Import JSON file mongoimport --db mydb --collection products --file products.json --jsonArray // Verify import mongo use mydb db.products.count() db.products.findOne()

תרגיל 2: Queries בסיסיות

מטרה: תרגול queries עם operators שונים

בנה שאילתות לענות על הדברים הבאים:

א. החזר רשומות עם type = "service"

db.products.find({ type: "service" })

ב. מספר פריטים עם price נמוך מ-100

db.products.find({ price: { $lt: 100 } }).count() // או db.products.countDocuments({ price: { $lt: 100 } })

ג. פריטים שה-name מכיל "AC3" ו-brand = "ACME"

db.products.find({ name: /AC3/, brand: "ACME" })

ד. פריטים עם warranty_years > 0 ו-price < 100

db.products.find({ warranty_years: { $gt: 0 }, price: { $lt: 100 } }).count()

תרגיל 3: עבודה עם Aviation Data + Indexes

מטרה: הבנת החשיבות של indexes לביצועים

שלב 1: ייבוא הנתונים

// Import CSV file mongoimport --db mydb --collection aviation \ --type csv --headerline --file Aviation_data.csv

שלב 2: בדוק שהנתונים נטענו

db.aviation.findOne() db.aviation.count()

שלב 3: מספר תאונות בארה"ב (ללא index)

db.aviation.find({ Country: "United States" }).count()

שלב 4: כמה documents נסרקו? (ללא index)

db.aviation.find({ Country: "United States" }).explain("executionStats") // Look for: // - totalDocsExamined (כמה documents נסרקו) // - executionTimeMillis (זמן ביצוע) // - stage: "COLLSCAN" (collection scan - לא טוב!)

שלב 5: הוסף index על Country

db.aviation.createIndex({ Country: 1 })

שלב 6: בדוק שוב - כמה documents נסרקו? (עם index)

db.aviation.find({ Country: "United States" }).explain("executionStats") // עכשיו תראה: // - totalDocsExamined הרבה יותר קטן! // - stage: "IXSCAN" (index scan - טוב!) // - executionTimeMillis הרבה יותר נמוך!

שלב 7: רשימת כל ה-indexes

db.aviation.getIndexes() // Output יראה: // [ // { v: 2, key: { _id: 1 }, name: "_id_" }, // Default // { v: 2, key: { Country: 1 }, name: "Country_1" } // שלנו! // ]

תרגיל 4: Compound Index

מטרה: למד איך compound index משפר queries עם מספר שדות

אופטימיזציה של queries על שני שדות:

// Create compound index db.aviation.createIndex({ Aircraft_Category: 1, Broad_Phase_of_Flight: 1 }) // Query using both fields db.aviation.find({ Aircraft_Category: "Airplane", Broad_Phase_of_Flight: "APPROACH" }) // Verify optimization db.aviation.find({ Aircraft_Category: "Airplane", Broad_Phase_of_Flight: "APPROACH" }).explain("executionStats") // Index תומך גם בqueries רק על Aircraft_Category // אבל לא רק על Broad_Phase_of_Flight (סדר חשוב!)

תרגיל 5: יצירה ועדכון - Authors Collection

מטרה: תרגול insert, update, ועבודה עם arrays

צור collection של מחברים ועדכן אותם:

// Create new database and collection use bookstore // Insert authors with genres array db.authors.insertMany([ { name: "Isaac Asimov", genres: ["Science Fiction", "Mystery", "Essay"], books: 500, born: 1920 }, { name: "Agatha Christie", genres: ["Mystery", "Crime", "Romance"], books: 66, born: 1890 } ]) // Update single document - תקן מספר ספרים db.authors.updateOne( { name: "Isaac Asimov" }, { $set: { books: 506 } } ) // Update multiple documents - סמן קלאסיקות db.authors.updateMany( { born: { $lt: 1900 } }, { $set: { classic: true } } ) // Add to array - הוסף ז'אנר db.authors.updateOne( { name: "Agatha Christie" }, { $push: { genres: "Thriller" } } ) // Verify db.authors.find().pretty()

תרגיל 6: מחיקה ונקיון

⚠️ זהירות: פעולות מחיקה הן בלתי הפיכות!

תרגיל זה מלמד איך לנקות databases ו-collections.

// Delete all documents in collection db.authors.deleteMany({}) // Collection עדיין קיים, רק ריק // Drop collection (מחיקה מלאה) db.authors.drop() // Drop database (מחיקת כל הdatabase!) use bookstore db.dropDatabase() // Verify show dbs // bookstore לא צריך להיות ברשימה

תרגיל 7: Aggregation Pipeline

מטרה: למד aggregation לניתוח נתונים מתקדם

חזור ל-products collection וחשב סטטיסטיקות:

use mydb // 1. Average price per type db.products.aggregate([ { $group: { _id: "$type", avgPrice: { $avg: "$price" }, count: { $sum: 1 } } }, { $sort: { avgPrice: -1 } } ]) // 2. Count products by brand db.products.aggregate([ { $group: { _id: "$brand", totalProducts: { $sum: 1 }, totalValue: { $sum: "$price" } } } ]) // 3. Products with warranty > 0, grouped by type db.products.aggregate([ { $match: { warranty_years: { $gt: 0 } } }, { $group: { _id: "$type", count: { $sum: 1 }, maxWarranty: { $max: "$warranty_years" } } } ])

🔧 פתרון בעיות ב-MongoDB

❌ Connection Refused / Cannot Connect

סימפטומים:

  • Error: "connection refused"
  • Cannot connect to MongoDB
  • Timeout errors

✅ פתרונות:

1. בדוק ש-MongoDB רץ:
// Check MongoDB process ps aux | grep mongod // Check MongoDB status (systemd) sudo systemctl status mongod // With Docker docker ps | grep mongo
2. בדוק Port:
// Check if MongoDB listening on port 27017 netstat -tlnp | grep 27017 sudo lsof -i :27017 // Test connection telnet localhost 27017
3. בדוק Configuration:
// Check mongod.conf cat /etc/mongod.conf # Look for: net: port: 27017 bindIp: 0.0.0.0 # Listen on all interfaces # Or bindIp: localhost # Only local connections
4. בדוק Logs:
// MongoDB logs location tail -f /var/log/mongodb/mongod.log // Docker logs docker logs mongodb

🐌 Queries איטיות

סימפטומים:

  • Queries לוקחות זמן רב
  • CPU usage גבוה
  • תגובה איטית של Application

✅ Debug & Fix:

1. Analyze Query עם explain():
// Check query execution db.collection.find({ field: value }).explain("executionStats") // Look for: // - executionTimeMillis (זמן) // - totalDocsExamined (כמה documents נסרקו) // - totalKeysExamined (כמה index keys) // - executionStages.stage (COLLSCAN = רע, IXSCAN = טוב)
2. הוסף Index:
// Create index on frequently queried fields db.collection.createIndex({ field: 1 }) // Compound index for multi-field queries db.collection.createIndex({ field1: 1, field2: -1 })
3. Enable Profiling:
// Log slow queries (>100ms) db.setProfilingLevel(1, { slowms: 100 }) // View slow queries db.system.profile.find().sort({ ts: -1 }).limit(10).pretty()
4. אופטימיזציות:
// Use projection (select only needed fields) db.users.find( { age: { $gt: 25 } }, { name: 1, email: 1, _id: 0 } // Only these fields ) // Limit results db.users.find().limit(100) // Use covered queries (index contains all fields) db.users.createIndex({ name: 1, email: 1 }) db.users.find( { name: "John" }, { name: 1, email: 1, _id: 0 } ).hint({ name: 1, email: 1 })

💾 Disk Space Issues

סימפטומים:

  • MongoDB crashes
  • Cannot write to database
  • "No space left on device"

✅ פתרונות:

1. בדוק Disk Usage:
// Check disk space df -h // Check MongoDB data size db.stats() db.collection.stats()
2. נקה Databases/Collections:
// Drop unused databases use oldDatabase db.dropDatabase() // Drop old collections db.oldCollection.drop() // Remove old documents db.logs.deleteMany({ createdAt: { $lt: new Date("2023-01-01") } })
3. Compact Collections:
// Reclaim disk space (blocks reads/writes!) db.runCommand({ compact: "collectionName" }) // Or repair database (longer, more thorough) db.repairDatabase()

🔴 Replica Set Issues

בעיות נפוצות:

  • Secondary not syncing
  • No Primary elected
  • Replica lag

✅ Debug Steps:

1. בדוק Status:
// Check replica set status rs.status() // Check configuration rs.conf() // Check replication lag rs.printReplicationInfo() rs.printSlaveReplicationInfo()
2. פתרונות:
// Reconfigure replica set rs.reconfig(config) // Step down primary (force new election) rs.stepDown() // Re-sync secondary (remove and add back) rs.remove("mongodb2.example.com:27017") rs.add("mongodb2.example.com:27017")

🔍 Debug Checklist

  1. ✅ MongoDB process רץ?
  2. ✅ Port 27017 פתוח?
  3. ✅ bindIp מוגדר נכון?
  4. ✅ Disk space פנוי?
  5. ✅ RAM מספיק?
  6. ✅ Indexes קיימים לqueries?
  7. ✅ Replica set healthy?

📋 Quick Reference

🔍 Query Operators

Operator תיאור דוגמה
$eq שווה ל { age: { $eq: 25 } }
$ne לא שווה ל { status: { $ne: "inactive" } }
$gt / $gte גדול מ / גדול או שווה { price: { $gt: 100 } }
$lt / $lte קטן מ / קטן או שווה { age: { $lt: 30 } }
$in נמצא ברשימה { status: { $in: ["A", "B"] } }
$nin לא נמצא ברשימה { status: { $nin: ["X", "Y"] } }
$exists שדה קיים { email: { $exists: true } }
$type טיפוס שדה { age: { $type: "number" } }
$regex Regular expression { name: { $regex: "^A" } }

📝 Update Operators

Operator תיאור דוגמה
$set הגדר ערך שדה { $set: { age: 30 } }
$unset הסר שדה { $unset: { temp: "" } }
$inc הוסף מספר (increment) { $inc: { views: 1 } }
$mul הכפל ב { $mul: { price: 1.1 } }
$rename שנה שם שדה { $rename: { "old": "new" } }
$min / $max עדכן רק אם קטן/גדול יותר { $min: { score: 50 } }
$currentDate הגדר לתאריך נוכחי { $currentDate: { updated: true } }
$push הוסף לarray { $push: { tags: "new" } }
$pull הסר מarray { $pull: { tags: "old" } }
$addToSet הוסף לarray (ייחודי) { $addToSet: { tags: "tag" } }

📊 Aggregation Pipeline Stages

Stage תיאור דוגמה
$match סינון documents { $match: { age: { $gt: 25 } } }
$project בחירת שדות { $project: { name: 1, age: 1 } }
$group קיבוץ לפי field { $group: { _id: "$city", total: { $sum: 1 } } }
$sort מיון { $sort: { age: -1 } }
$limit הגבלת תוצאות { $limit: 10 }
$skip דילוג על תוצאות { $skip: 5 }
$lookup Join עם collection אחר { $lookup: { from: "orders", localField: "_id", foreignField: "userId", as: "userOrders" } }
$unwind פירוק array { $unwind: "$tags" }

🎯 Common Patterns

Pagination:

// Page 1 (skip 0, limit 10) db.products.find().skip(0).limit(10) // Page 2 (skip 10, limit 10) db.products.find().skip(10).limit(10) // Better for large collections - use range query on _id db.products.find({ _id: { $gt: lastSeenId } }).limit(10)

Search Pattern:

// Case-insensitive search db.products.find({ name: { $regex: "laptop", $options: "i" } }) // Starts with db.products.find({ name: /^AC/ }) // Contains db.products.find({ name: /pattern/ }) // Text search (requires text index) db.products.createIndex({ name: "text", description: "text" }) db.products.find({ $text: { $search: "mongodb tutorial" } })

Upsert Pattern:

// Update if exists, insert if not db.users.updateOne( { email: "user@example.com" }, { $set: { name: "User", lastLogin: new Date() } }, { upsert: true } )

✨ MongoDB Best Practices

🏗️ Schema Design

✅ Best Practices:

  • Embed vs Reference:
    • Embed: נתונים שנקראים ביחד (1-to-1, 1-to-few)
    • Reference: נתונים עצמאיים או 1-to-many רב
  • הימנע מdocuments גדולים מדי: max 16MB
  • שקול את access patterns: איך האפליקציה קוראת את הנתונים?
  • Denormalization: כפילות מותרת אם משפרת performance

דוגמה - Embed vs Reference:

// ✅ Embed - 1-to-few (user addresses) { _id: 1, name: "John", addresses: [ { street: "123 Main St", city: "NYC" }, { street: "456 Park Ave", city: "LA" } ] } // ✅ Reference - 1-to-many (user orders) // Users collection { _id: 1, name: "John" } // Orders collection { _id: 101, userId: 1, total: 50.00 } { _id: 102, userId: 1, total: 75.00 }

🔍 Indexing Best Practices

  • צור indexes לqueries תכופים (בדוק עם explain)
  • ESR Rule: Equality, Sort, Range - סדר השדות ב-compound index
  • הימנע מיותר מדי indexes: מאט writes!
  • Use covered queries: index מכיל את כל השדות הדרושים
  • מחק indexes לא בשימוש
  • נטר index usage: $indexStats

ESR Rule דוגמה:

// Query: Find active users, sorted by age, in age range db.users.find({ status: "active", // Equality age: { $gte: 25, $lte: 40 } // Range }).sort({ age: 1 }) // Sort // Optimal index (E-S-R order): db.users.createIndex({ status: 1, // E - Equality first age: 1 // S + R - Sort and Range together })

⚡ Performance Optimization

Write Performance:

  • השתמש ב-Bulk Operations לכמויות גדולות
  • הפחת מספר indexes (כל index = overhead)
  • השתמש ב-Unordered Bulk לparallel writes
  • שקול Write Concern נמוך יותר (w:1) אם לא קריטי

Read Performance:

  • צור indexes לqueries תכופים
  • השתמש ב-projection (רק שדות נדרשים)
  • השתמש ב-limit() להגבלת תוצאות
  • Covered queries - הכי מהירים!
  • שקול Read Preference לsecondaries

🔒 Security Best Practices

⚠️ Production Security:

  • ☐ Authentication: enable auth, create users
  • ☐ Authorization: roles בעלי הרשאות מינימליות
  • ☐ Encryption in Transit: SSL/TLS
  • ☐ Encryption at Rest: encrypted storage engine
  • ☐ Network Security: bindIp, firewall
  • ☐ Audit Logging: עקוב אחרי פעילות
  • ☐ Regular Backups: mongodump / snapshots
  • ☐ Updates: תמיד גרסה עדכנית

💾 Backup & Recovery

// mongodump - backup entire database mongodump --db mydb --out /backup/ // mongorestore - restore database mongorestore --db mydb /backup/mydb/ // Backup specific collection mongodump --db mydb --collection users --out /backup/ // Backup with query mongodump --db mydb --collection logs \ --query '{"createdAt": {"$gt": {"$date": "2024-01-01"}}}' // Export to JSON mongoexport --db mydb --collection users --out users.json // Import from JSON mongoimport --db mydb --collection users --file users.json
