🎯 מבוא ל-Elastic Stack

מהו Elastic Stack?

Elastic Stack (המכונה גם ELK Stack) הוא אוסף כלים לניהול, עיבוד, אחסון וניתוח נתונים בזמן אמת.

הרכיבים העיקריים

🔍 Elasticsearch

מנוע חיפוש וניתוח מבוזר המאפשר אחסון, חיפוש וניתוח כמויות גדולות של נתונים בזמן אמת.

📊 Kibana

ממשק ויזואלי לחיפוש, צפייה וניתוח הנתונים האגורים ב-Elasticsearch.

⚙️ Logstash

צינור עיבוד נתונים לאיסוף, עיבוד והעברת נתונים מ מקורות שונים ל-Elasticsearch.

📦 Beats

סוכנים קלי משקל לשליחת נתונים ממאות או אלפי מכונות ל-Logstash או Elasticsearch.

תרחישי שימוש נפוצים

  • ניתוח לוגים - ריכוז וניתוח לוגים מכל המערכות בארגון
  • Application Performance Monitoring (APM) - מעקב אחר ביצועי אפליקציות
  • Security Analytics - זיהוי איומים וניתוח אבטחה
  • Business Analytics - ניתוח נתונים עסקיים בזמן אמת
  • Full-Text Search - חיפוש טקסט מלא באתרים ואפליקציות

🔍 מהו Elasticsearch?

Elasticsearch הוא מנוע חיפוש וניתוח מבוזר, בנוי על Apache Lucene, המאפשר אחסון, חיפוש וניתוח נתונים בזמן כמעט-אמיתי (Near Real-Time).

מאפיינים עיקריים

🌐 מבוזר (Distributed)

Elasticsearch בנוי להיות מבוזר מההתחלה. הוא מאפשר להפיץ נתונים על פני מספר servers (nodes) ולהרחיב את המערכת אופקית.

⚡ Near Real-Time (NRT)

יש זמן השהיה קטן (בדרך כלל שנייה אחת) מרגע שאתה מאנדקס document עד שהוא הופך ניתן לחיפוש.

📝 Schema-Free

אין צורך להגדיר schema מראש. Elasticsearch יוצר אוטומטית mapping על בסיס המבנה של ה-documents שנשלחים.

🔄 RESTful API

כל הפעולות מתבצעות דרך REST API פשוט, באמצעות JSON over HTTP.

💡 למה Elasticsearch כל כך מהיר?

הסוד נמצא במבנה ה-Inverted Index שמאפשר חיפוש מהיר ביותר אפילו על מיליארדי documents.

📦 ELK Stack - תהליך עבודה מלא

1️⃣ Beats - איסוף נתונים

🎵 מהם Beats?

Beats הם data shippers קלים שאתה מתקין כ-agents על השרתים שלך לשליחת נתונים תפעוליים.

סוגי Beats פופולריים:

  • Filebeat - לוגים מקבצים
  • Metricbeat - metrics מערכת ושירותים
  • Packetbeat - נתוני רשת
  • Heartbeat - uptime monitoring
  • Auditbeat - audit data

2️⃣ Logstash - עיבוד נתונים

⚙️ מה עושה Logstash?

Logstash הוא data processing pipeline שמקבל נתונים, מעבד אותם (parsing, filtering, enrichment) ושולח אותם ל-Elasticsearch.

שלבי Logstash Pipeline:

  • Input - קבלת נתונים ממקורות שונים
  • Filter - עיבוד ושינוי הנתונים (grok, mutate, date)
  • Output - שליחת הנתונים המעובדים ליעד (Elasticsearch)

3️⃣ Elasticsearch - אחסון וחיפוש

הנתונים המעובדים נשמרים ב-Elasticsearch במבנה אופטימלי לחיפוש מהיר.

4️⃣ Kibana - ויזואליזציה וניתוח

📊 יכולות Kibana:

  • Visualize - יצירת תרשימים וגרפים
  • Dashboard - צירוף visualizations ל-dashboards
  • Canvas - יצירת infographics

🏗️ ארכיטקטורת Elasticsearch

מבנה Cluster

Cluster הוא אוסף של nodes (שרתים) שעובדים יחד ומחזיקים את כל הנתונים שלך.

כל cluster מזוהה על ידי שם ייחודי (ברירת מחדל: "elasticsearch").

🖥️ סוגי Nodes

Master Node

אחראי על פעולות ניהול ברמת ה-cluster: יצירת/מחיקת indices, מעקב אחר nodes, הקצאת shards.

Data Node

מחזיק data ומבצע פעולות הקשורות לנתונים: CRUD, search, aggregations.

Ingest Node

מבצע pre-processing על documents לפני ה-indexing.

Coordinating Node

מנתב requests, מבצע reduce phase של search, מפזר bulk indexing.

Index, Document, Field

📊 אנלוגיה למסד נתונים:

  • Index ≈ Database
  • Document ≈ Row/Record
  • Field ≈ Column

Index

אוסף של documents עם מאפיינים דומים. לדוגמה: index של logs, index של users.

Document

יחידת המידע הבסיסית שניתן לאנדקס. זה JSON object.

Field

מפתח-ערך בתוך document. לכל field יש data type.

🗂️ Sharding & Replication

למה צריך Sharding?

Index יכול להכיל כמות גדולה של נתונים שעלולה לעבור את מגבלות החומרה של node בודד.

Sharding מאפשר לפצל את ה-index למספר חלקים (shards) שיכולים להיות מאוכסנים על nodes שונים.

✅ יתרונות Sharding:

  • Scale horizontally - הוספת יותר נתונים מעבר ליכולת של node בודד
  • Parallelism - פעולות (חיפוש, indexing) יכולות לרוץ במקביל על shards שונים
  • Performance - חלוקת העומס על nodes מרובים

Primary vs Replica Shards

Primary Shard

כל document שייך ל-primary shard אחד. מספר ה-primary shards נקבע בעת יצירת ה-index.

Replica Shard

עותק של primary shard. מספק high availability במקרה של כשל, ומאפשר להגדיל את throughput של queries.

🎯 Replica Shards - למה?

  • High Availability - אם node נופל, הנתונים עדיין זמינים
  • Scalability - search queries יכולים לרוץ על כל הרפליקות במקביל

⚠️ כלל חשוב:

Replica shard לעולם לא יוקצה לאותו node כמו ה-primary shard שלו (למניעת data loss).

⚡ Inverted Index - הקסם מאחורי המהירות

Inverted Index הוא מבנה הנתונים המרכזי שעושה את Elasticsearch כל כך מהיר.

איך זה עובד?

במקום לשמור "document → מילים", Elasticsearch שומר "מילה → documents".

דוגמה:

Doc 1: "The quick brown fox"

Doc 2: "The quick blue rabbit"

Doc 3: "The slow brown turtle"

Inverted Index:

  • the → [1, 2, 3]
  • quick → [1, 2]
  • brown → [1, 3]
  • fox → [1]
  • blue → [2]
  • rabbit → [2]
  • slow → [3]
  • turtle → [3]

עכשיו כשאתה מחפש "brown", Elasticsearch יודע מיד שזה מופיע ב-documents 1 ו-3 - בלי לעבור על כל ה-documents!

תהליך Indexing

  1. Analysis - הטקסט מפוצל למילים (tokens)
  2. Normalization - lowercase, הסרת ניקוד וכו'
  3. Indexing - יצירת ה-inverted index

📄 Documents & Fields

מהו Document?

Document הוא יחידת המידע הבסיסית ב-Elasticsearch. זה JSON object שמכיל fields.

דוגמת Document:

{ "_index": "users", "_id": "1", "_source": { "name": "Lev Beck", "email": "lev@example.com", "age": 30, "joined": "2024-01-15", "interests": ["data", "elasticsearch", "coding"] } }

Metadata Fields

  • _index - איזה index ה-document שייך אליו
  • _id - מזהה ייחודי של ה-document
  • _source - ה-JSON המקורי שנשלח
  • _score - relevance score בחיפוש

Field Data Types

טקסט

  • text - full-text search
  • keyword - exact values

מספרים

  • long, integer, short, byte
  • double, float

תאריכים ובוליאנים

  • date
  • boolean

מורכבים

  • object - JSON object
  • nested - array of objects

🗺️ Mappings

Mapping מגדיר איך documents ו-fields שלהם מאוחסנים ומאונדקסים.

זה בערך כמו schema במסד נתונים רלציוני.

Dynamic Mapping

Elasticsearch יכול להסיק אוטומטית את ה-mapping על בסיס הנתונים שנשלחים:

  • String → text + keyword
  • Number → long/double
  • Boolean → boolean
  • Date string → date

Explicit Mapping

אתה יכול (ומומלץ בפרודקשן) להגדיר mapping מפורש:

PUT /my-index { "mappings": { "properties": { "title": { "type": "text" }, "author": { "type": "keyword" }, "pages": { "type": "integer" }, "published": { "type": "date" } } } }

⚠️ חשוב לדעת:

לא ניתן לשנות mapping של field קיים! צריך ליצור index חדש ולעשות reindex.

🔎 סוגי Queries ב-Elasticsearch

Query Context vs Filter Context

Query Context

עונה על: "כמה טוב ה-document מתאים?"

מחשב relevance score (_score)

Filter Context

עונה על: "האם ה-document מתאים?"

תשובה: כן/לא (בלי score)

מהיר יותר ו-cacheable

Full-Text Queries

Match Query

החיפוש הבסיסי ביותר. מבצע analysis על הטקסט.

POST /books/_search { "query": { "match": { "title": "elasticsearch guide" } } }

Multi-Match Query

חיפוש במספר fields במקביל.

Match Phrase

חיפוש ביטוי מדויק (המילים בסדר הנכון).

Term-Level Queries

חיפוש ערכים מדויקים (ללא analysis):

  • Term - ערך מדויק אחד
  • Terms - מספר ערכים (OR)
  • Range - טווח (תאריכים, מספרים)
  • Exists - בדיקה שה-field קיים
  • Wildcard - תווים כללים (* ?)
  • Regexp - regular expressions

Boolean Query

שילוב של queries עם לוגיקה בוליאנית:

  • must - חייב להתאים (AND) + משפיע על ה-score
  • should - רצוי (OR) + משפיע על ה-score
  • must_not - לא צריך להתאים (NOT)
  • filter - חייב להתאים אבל בלי score

📊 Aggregations - ניתוח נתונים

Aggregations מאפשרות לנו לבצע ניתוח על הנתונים - סכומים, ממוצעים, קיבוצים ועוד.

סוגי Aggregations

1. Metric Aggregations

חישובים מתמטיים על ערכים:

  • avg - ממוצע
  • sum - סכום
  • min / max - מינימום / מקסימום
  • stats - כל הנ"ל ביחד
  • cardinality - ספירת ערכים ייחודיים

2. Bucket Aggregations

קיבוץ documents ל-"דליים":

  • terms - קיבוץ לפי ערכים (top N)
  • range - קיבוץ לפי טווחים
  • date_histogram - קיבוץ לפי תאריכים
  • histogram - קיבוץ לפי מספרים

דוגמה - ספירת לוגים לפי רמת חומרה:

POST /logs/_search { "size": 0, "aggs": { "levels": { "terms": { "field": "level.keyword" } } } }

3. Pipeline Aggregations

aggregations על תוצאות של aggregations אחרים.

✨ Best Practices

🏗️ תכנון Index

מספר Shards

  • יותר מדי shards = overhead
  • מעט מדי = bottleneck
  • כלל אצבע: shard size בין 10GB-50GB
  • למידע קטן: 1-2 shards מספיק

Replicas

  • מינימום 1 replica לפרודקשן (high availability)
  • יותר replicas = יותר search throughput
  • אבל: דורש יותר אחסון ויותר indexing overhead

📝 Mappings

  • הגדר mapping מפורש לפרודקשן (אל תסמוך על dynamic mapping)
  • השתמש ב-keyword לשדות שאתה לא צריך full-text search עליהם
  • השבת _all field אם לא צריך (מיושן ממילא)
  • השתמש ב-index: false לשדות שלא צריך לחפש בהם

🔍 Queries

⚡ ביצועים:

  • השתמש ב-filter context כשאין צורך ב-scoring (cacheable!)
  • הגבל את מספר התוצאות עם size
  • השתמש ב-_source filtering כדי להחזיר רק שדות שצריך
  • העדף term queries על match כשמחפשים ערכים מדויקים

🗂️ Index Lifecycle

עבור נתונים time-based (לוגים):

  • צור indices חדשים לפי זמן (daily/weekly)
  • השתמש ב-Index Lifecycle Management (ILM)
  • העבר indices ישנים ל-cold/frozen tier
  • מחק או ארכב indices ישנים מדי

🔐 Security

  • אל תחשוף Elasticsearch ישירות לאינטרנט
  • השתמש ב-authentication & authorization
  • הגדר HTTPS/TLS
  • גבה (backup) את הנתונים באופן קבוע

📊 Monitoring

  • עקוב אחר cluster health (green/yellow/red)
  • נטר disk space, memory, CPU
  • בדוק slow queries
  • השתמש ב-Elastic Stack Monitoring

הקמה יצירת Index והוספת נתונים

צור index בשם bookdb_index עם 1 shard, והוסף 4 ספרים באמצעות _bulk API.

PUT /bookdb_index { "settings": { "number_of_shards": 1 } } POST /bookdb_index/_doc/_bulk {"index":{"_id":1}} {"title":"Elasticsearch: The Definitive Guide","authors":["clinton gormley","zachary tong"],"summary":"A distributed real-time search and analytics engine","publish_date":"2015-02-07","num_reviews":20,"publisher":"oreilly"} {"index":{"_id":2}} {"title":"Taming Text","authors":["grant ingersoll","thomas morton"],"summary":"organize text using full-text search","publish_date":"2013-01-24","num_reviews":12,"publisher":"manning"} {"index":{"_id":3}} {"title":"Elasticsearch in Action","authors":["radu gheorge","matthew lee hinman"],"summary":"build scalable search applications","publish_date":"2015-12-03","num_reviews":18,"publisher":"manning"} {"index":{"_id":4}} {"title":"Solr in Action","authors":["trey grainger","timothy potter"],"summary":"Comprehensive guide to Apache Solr","publish_date":"2014-04-05","num_reviews":23,"publisher":"manning"}
01 החזר את כל ה-documents
GET /bookdb_index/_search
02 חיפוש Multi-Match

חפש את המילה "guide" בכל השדות.

POST /bookdb_index/_search { "query": { "multi_match": { "query": "guide" } } }
03 חיפוש בשדה ספציפי

חפש את הביטוי "in action" בשדה title.

POST /bookdb_index/_search { "query": { "match": { "title": "in action" } } }
04 Boosting - הגברת רלוונטיות

חפש "search analytics" עם boost של 3.

POST /bookdb_index/_search { "query": { "multi_match": { "query": "search analytics", "boost": 3 } } }
05 לוגיקה בוליאנית

מצא ספרים עם "Elasticsearch" או "Solr" בכותרת, שנכתבו על ידי "Clinton Gormley", אבל לא "Radu Gheorge".

POST /bookdb_index/_search { "query": { "bool": { "must": { "bool": { "should": [ {"match": {"title": "Elasticsearch"}}, {"match": {"title": "Solr"}} ], "must": {"match": {"authors": "clinton gormley"}} } }, "must_not": {"match": {"authors": "radu gheorge"}} } } }
06 Wildcard & Highlight

מצא מחברים ששמם מתחיל ב-'t' והדגש את שדה המחברים.

POST /bookdb_index/_search { "query": { "wildcard": { "authors": "t*" } }, "highlight": { "fields": { "authors": {} } } }
07 Regular Expression

מצא מחברים ששמם מתחיל ב-'t' ומסתיים ב-'y'.

POST /bookdb_index/_search { "query": { "regexp": { "authors": "t[a-z]*y" } } }
08 Range Query

חפש ספרים שפורסמו ב-2015 בלבד.

POST /bookdb_index/_search { "query": { "range": { "publish_date": { "gte": "2015-01-01", "lte": "2015-12-31" } } } }
09 סינון תוצאות

ספרים עם "Elasticsearch" בכותרת/תקציר ו-20+ ביקורות.

POST /bookdb_index/_search { "query": { "bool": { "must": { "multi_match": { "query": "elasticsearch", "fields": ["title", "summary"] } }, "filter": { "range": { "num_reviews": {"gte": 20} } } } } }
