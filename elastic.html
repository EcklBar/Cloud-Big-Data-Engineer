Elastic Stack (לשעבר ELK Stack) הוא אוסף של כלים קוד פתוח לאיסוף, עיבוד, אחסון, חיפוש, ניתוח וויזואליזציה של נתונים בזמן אמת.

מהו Search Engine Database?

Search Engine Database הוא סוג של מסד נתונים NoSQL המיועד לחיפוש תוכן

  • משתמש ב-indexes כדי לסווג מאפיינים דומים בין נתונים
  • מותאם לעבודה עם נתונים ארוכים, חצי-מובנים או לא-מובנים
  • מציע שיטות מיוחדות כמו Full-Text Search
  • תמיכה ב-complex search expressions ו-scoring/ranking

מהו Full-Text Search?

Full-Text Search מתייחס לטכניקות לחיפוש במסמך או באוסף מסמכים

  • מנוע החיפוש בודק את כל המילים בכל מסמך מאוחסן
  • מנסה להתאים לקריטריוני החיפוש (טקסט שצוין על ידי המשתמש)
  • Full-Text Search מחולק לשני משימות: INDEXING ו-SEARCHING

🔎 Apache Lucene & Inverted Index

מהו Apache Lucene?

Apache Lucene היא ספריית מנוע חיפוש קוד פתוח

  • נכתב במקור ב-Java על ידי Doug Cutting
  • מספק ספריית Full-Text Search
  • מקל על הוספת פונקציונליות חיפוש לאפליקציות
  • משיג תגובות חיפוש מהירות באמצעות Inverted Index

מהו Inverted Index?

Inverted Index מאחסן מיפוי הפוך: מערכים לדוקומנטים

במקום לחפש בטקסט ישירות, חיפוש ב-index הרבה יותר מהיר!

דוגמה: Forward vs Inverted Index

📄 Documents לדוגמה

// Document 1 { "id": 1, "job_title": "Software Engineer", "desc": "software engineer at a great company", "skills": ".Net, C#, java" } // Document 2 { "id": 2, "job_title": "Registered Nurse", "desc": "a registered nurse at hospital doing hard work", "skills": "oncology, phlebotomy" } // Document 3 { "id": 3, "job_title": "Java Developer", "desc": "a software engineer or a java engineer doing work", "skills": "java, scala, hibernate" }

📑 Forward Index (Docs → Terms)

מיפוי מ-Document לערכים:

  • Doc 1 → [software, engineer, at, a, great, company]
  • Doc 2 → [a, registered, nurse, at, hospital, doing, hard, work]
  • Doc 3 → [a, software, engineer, or, a, java, engineer, doing, work]

🔍 Inverted Index (Terms → Docs)

מיפוי מערך ל-documents:

  • software → [Doc 1, Doc 3]
  • engineer → [Doc 1, Doc 3, Doc 3]
  • java → [Doc 1, Doc 3]
  • nurse → [Doc 2]
  • work → [Doc 2, Doc 3]

🚀 עכשיו חיפוש אחר "java engineer" מיידי - רק בודקים ב-index!

💡 למה זה מהיר יותר?

במקום לסרוק את כל ה-documents (O(n)), חיפוש ב-inverted index הוא O(1)!

🏗️ ארכיטקטורת Elasticsearch

Cluster Architecture

Elasticsearch Cluster מורכב מ-nodes מסוגים שונים:

👑 Master Node

מנהל את ה-cluster: יצירת/מחיקת indices, tracking nodes

📊 Data Node

מחזיק נתונים ומבצע פעולות CRUD, search, aggregations

🔀 Coordinating Node

מנתב requests ומאגד תוצאות

🔍 Ingest Node

מעבד documents לפני אינדוקס

Shards & Replicas

Index מחולק ל-Shards לצורך סקלביליות

  • Primary Shard - הנתונים המקוריים
  • Replica Shard - עותק לגיבוי וביצועים
  • כל shard הוא Lucene index עצמאי
  • ברירת מחדל: 1 primary + 1 replica

Deployment Model

מודל פריסה טיפוסי:

  • 🔹 Development - Node בודד
  • 🔸 Production - מינימום 3 nodes (high availability)
  • 🔶 Large Scale - עשרות או מאות nodes
  • נפרד nodes לפי תפקידים (master, data, ingest)

⚠️ Best Practices:

  • תמיד השתמש במספר replicas בייצור
  • הפרד master nodes מ-data nodes בסביבות גדולות
  • השתמש ב-dedicated coordinating nodes לעומסים גבוהים

🔬 Analysis in Elasticsearch - Analyzers

מהם Analyzers?

Analyzer הוא תהליך שמעבד טקסט לפני אינדוקס או חיפוש

  • רק שדות מסוג text מנותחים (analyzed)
  • שדות מסוג keyword לא מנותחים (indexed as-is)
  • חשוב לחיפוש full-text יעיל

רכיבי Analyzer

Character
Filters
→
Tokenizer
→
Token
Filters

1️⃣ Character Filters

מקבלים טקסט מקורי ויכולים לשנות אותו

  • הוספת, הסרת או שינוי תווים
  • דוגמה: הסרת HTML markup
  • עובדים על character stream

2️⃣ Tokenizer

מפצל את הטקסט ל-tokens בודדים

  • Tokens הם בדרך כלל מילים
  • משפט עם 10 מילים → מערך של 10 tokens
  • Analyzer יכול להכיל רק tokenizer אחד
  • ברירת מחדל: Standard Tokenizer

3️⃣ Token Filters

מוסיפים, מסירים או משנים tokens

  • lowercase - המרה לאותיות קטנות
  • stop - הסרת stop words (the, is, at)
  • synonym - הוספת מילים נרדפות
  • stemmer - צמצום למילת שורש

Analyze API

Analyze API מאפשר לבדוק איך analyzer מטפל בטקסט

// Test standard analyzer POST /_analyze { "analyzer": "standard", "text": "I'm in the mood for drinking semi-dry red wine!" } // Test specific tokenizer POST /_analyze { "tokenizer": "standard", "text": "Hello World! This is Elasticsearch." } // Test with filters POST /_analyze { "tokenizer": "standard", "filter": ["lowercase", "stop"], "text": "The Quick Brown Fox Jumps" } // Test custom analyzer POST /my-index/_analyze { "analyzer": "my_custom_analyzer", "text": "Testing custom analysis" }

Built-in Analyzers

standard

ברירת מחדל, מפצל לפי רווחים ופיסוק

simple

מפצל לפי non-letters

whitespace

מפצל רק לפי רווחים

stop

כמו simple + מסיר stop words

keyword

לא מנתח, משאיר as-is

pattern

משתמש ב-regex לפיצול

Custom Analyzer

PUT /my-index { "settings": { "analysis": { "analyzer": { "my_custom_analyzer": { "type": "custom", "tokenizer": "standard", "char_filter": ["html_strip"], "filter": ["lowercase", "stop", "snowball"] } } } } }

🗺️ Mapping ב-Elasticsearch

מהו Mapping?

Mapping הוא התהליך של הגדרת מבנה השדות ב-document

  • דומה ל-Schema ב-RDBMS
  • מגדיר איך documents יאוחסנו ויאונדקסו
  • שדות text מנותחים בזמן אינדוקס
  • שדות keyword לא מנותחים (טובים ל-exact matches)

שימושים ל-Mapping

השתמש ב-mapping כדי להגדיר:

  • אילו שדות צריכים להיות full text
  • אילו שדות מכילים מספרים, תאריכים או geolocations
  • איך לאנדקס שדות מסוימים
  • פורמטים מותאמים אישית

Field Data Types

text

טקסט מלא, מנותח

keyword

טקסט לא מנותח

long, integer

מספרים שלמים

float, double

מספרים עשרוניים

boolean

true/false

date

תאריכים

object

JSON object מוטמע

nested

מערך של objects

geo_point

קואורדינטות

יצירת Mapping

PUT /inspections { "mappings": { "properties": { "business_address": { "type": "text" }, "business_city": { "type": "text", "fields": { "keyword": { "type": "keyword" } } }, "business_name": { "type": "text", "analyzer": "standard" }, "coordinates": { "type": "geo_point" }, "inspection_date": { "type": "date", "format": "yyyy-MM-dd" }, "violation_id": { "type": "keyword" }, "score": { "type": "integer" }, "active": { "type": "boolean" } } } }

הצגת Mapping

// Get mapping for specific index GET /inspections/_mapping // Get mapping for all indices GET /_all/_mapping // Get mapping for specific field GET /inspections/_mapping/field/business_city

עדכון Mapping

// Add new field to existing mapping PUT /inspections/_mapping { "properties": { "new_field": { "type": "text" } } }

⚠️ חשוב לדעת:

  • לא ניתן לשנות mapping של שדה קיים
  • צריך ליצור index חדש עם mapping מעודכן
  • ואז להעתיק נתונים עם Reindex API

Multi-Fields

PUT /my-index { "mappings": { "properties": { "city": { "type": "text", "fields": { "keyword": { "type": "keyword" }, "english": { "type": "text", "analyzer": "english" } } } } } } // Query: // city → analyzed text search // city.keyword → exact match // city.english → English analyzer

🎯 Scoring ב-Elasticsearch

מהו Scoring?

Scoring קובע כמה רלוונטי document לשאילתה

  • Score נקבע על סמך התאמות השדות לשאילתה
  • Results ממוינים לפי score (רלוונטיות)
  • Elasticsearch משתמש ב-Lucene's Practical Scoring Function
  • ציון גבוה יותר = רלוונטיות גבוהה יותר

תוצאת חיפוש עם Scores

{ "hits": { "total": { "value": 4, "relation": "eq" }, "max_score": 0.5022658, "hits": [ { "_index": "inspections", "_type": "_doc", "_id": "2", "_score": 0.5022658, "_source": { "business_address": "10 Mason St", "business_city": "San Francisco" } }, { "_index": "inspections", "_type": "_doc", "_id": "5", "_score": 0.3456789, "_source": { "business_address": "123 Main St", "business_city": "Oakland" } } ] } }

גורמים המשפיעים על Score

TF (Term Frequency)

כמה פעמים המונח מופיע

IDF (Inverse Document Frequency)

כמה נדיר המונח

Field Length

אורך השדה

Boost

הגברת חשיבות

שאילתות עם Boost

GET /my-index/_search { "query": { "bool": { "should": [ { "match": { "title": { "query": "elasticsearch", "boost": 2 } } }, { "match": { "content": "elasticsearch" } } ] } } } // title פי 2 יותר חשוב מ-content

Explain API

// Explain why document got specific score GET /my-index/_explain/1 { "query": { "match": { "message": "elasticsearch" } } }

Function Score Query

GET /my-index/_search { "query": { "function_score": { "query": { "match": { "message": "elasticsearch" } }, "functions": [ { "filter": { "match": { "status": "published" } }, "weight": 2 }, { "field_value_factor": { "field": "popularity", "factor": 1.2, "modifier": "sqrt" } } ], "score_mode": "sum", "boost_mode": "multiply" } } }

  • mongodb - שליחה ל-MongoDB
  • s3 - העלאה ל-S3
  • email - שליחת מייל
  • stdout - הדפסה למסך (לבדיקות)

דוגמת Configuration

# Input: Read from file input { file { path => "/var/log/apache/access.log" start_position => "beginning" sincedb_path => "/dev/null" } } # Filter: Parse Apache log filter { grok { match => { "message" => "%{COMBINEDAPACHELOG}" } } date { match => [ "timestamp", "dd/MMM/yyyy:HH:mm:ss Z" ] } geoip { source => "clientip" } mutate { convert => { "response" => "integer" } convert => { "bytes" => "integer" } } } # Output: Send to Elasticsearch output { elasticsearch { hosts => ["localhost:9200"] index => "apache-logs-%{+YYYY.MM.dd}" } stdout { codec => rubydebug } }

Grok Patterns

// Common patterns %{IP:client} # IP address %{WORD:action} # Word %{NUMBER:duration} # Number %{TIMESTAMP_ISO8601} # ISO timestamp // Example: Parse custom log %{IP:client} %{WORD:method} %{URIPATHPARAM:request} %{NUMBER:bytes} %{NUMBER:duration} // Log: 192.168.1.1 GET /api/users 1234 0.123 // Result: { "client": "192.168.1.1", "method": "GET", "request": "/api/users", "bytes": "1234", "duration": "0.123" }

💓 Beats - Lightweight Data Shippers

מהם Beats?

Beats הם כלים קלים לשליחת נתונים

  • כתובים ב-Go - קלים ומהירים
  • ממוקדים בסוג נתון בודד
  • שולחים ישירות ל-Elasticsearch או דרך Logstash
  • דורשים משאבים מינימליים

Filebeat Configuration

filebeat.inputs: - type: log enabled: true paths: - /var/log/nginx/access.log - /var/log/nginx/error.log fields: service: nginx environment: production # Output to Elasticsearch output.elasticsearch: hosts: ["localhost:9200"] index: "nginx-logs-%{+yyyy.MM.dd}" # Or output to Logstash output.logstash: hosts: ["localhost:5044"] # Enable modules filebeat.modules: - module: nginx access: enabled: true error: enabled: true

Metricbeat Configuration

metricbeat.modules: - module: system metricsets: - cpu - memory - network - filesystem enabled: true period: 10s - module: docker metricsets: - container - cpu - memory - network hosts: ["unix:///var/run/docker.sock"] period: 10s output.elasticsearch: hosts: ["localhost:9200"]

תרגיל 2: יצירת Index ו-Documents

// Create index with mapping PUT /products { "mappings": { "properties": { "name": { "type": "text" }, "price": { "type": "float" }, "category": { "type": "keyword" }, "description": { "type": "text" }, "stock": { "type": "integer" }, "created_at": { "type": "date" } } } } // Add documents POST /products/_doc { "name": "Laptop Pro", "price": 1299.99, "category": "Electronics", "description": "High-performance laptop", "stock": 50, "created_at": "2024-01-01" } POST /products/_doc { "name": "Running Shoes", "price": 89.99, "category": "Sports", "description": "Comfortable running shoes", "stock": 100, "created_at": "2024-01-02" }

תרגיל 3: שאילתות חיפוש

// Match query GET /products/_search { "query": { "match": { "description": "laptop" } } } // Range query GET /products/_search { "query": { "range": { "price": { "gte": 50, "lte": 1000 } } } } // Bool query GET /products/_search { "query": { "bool": { "must": [ { "match": { "category": "Electronics" } } ], "filter": [ { "range": { "price": { "gte": 500 } } } ] } } } // Aggregation GET /products/_search { "size": 0, "aggs": { "categories": { "terms": { "field": "category" } }, "avg_price": { "avg": { "field": "price" } } } }

תרגיל 4: בדיקת Analyzers

// Test standard analyzer POST /_analyze { "analyzer": "standard", "text": "The Quick Brown Fox Jumps Over the Lazy Dog!" } // Test lowercase filter POST /_analyze { "tokenizer": "standard", "filter": ["lowercase"], "text": "The Quick Brown Fox" } // Test stop filter POST /_analyze { "tokenizer": "standard", "filter": ["lowercase", "stop"], "text": "The quick brown fox jumps over the lazy dog" }

תרגיל 5: Index Management

// List all indices GET /_cat/indices?v // Get index settings GET /products/_settings // Get index mapping GET /products/_mapping // Update index settings PUT /products/_settings { "index": { "number_of_replicas": 2 } } // Create index alias POST /_aliases { "actions": [ { "add": { "index": "products", "alias": "my-products" } } } } // Reindex POST /_reindex { "source": { "index": "products" }, "dest": { "index": "products-v2" } } // Delete index DELETE /products
