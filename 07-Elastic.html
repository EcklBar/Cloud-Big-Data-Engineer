The Core Value

Why use this over a SQL DB? The Inverted Index.

Instead of scanning every row for "Error", Elastic flips the data. It stores "Error" and a list of every document ID that contains it. [cite_start]Search becomes instant[cite: 223].

Index Lifecycle Management (ILM)

In the real world, logging data is massive. You cannot keep 5 years of logs on your fastest, most expensive hardware. [cite_start]The PDF outlines a Hot-Warm-Cold architecture [cite: 416-433].

Phase Role Hardware Profile
🔥 HOT Ingesting new data. Heavy indexing and querying. High CPU, Fast SSD NVMe.
🍵 WARM Data is read-only (7-30 days old). Still queried often. Medium CPU, Standard SSD/HDD.
❄️ COLD Rarely queried (months/years old). For compliance. Low CPU, High-capacity HDD (Cheap storage).
🗑️ DELETE Data is purged automatically. N/A
Real World Tip: Set up an ILM Policy in Kibana to automatically "Rollover" indexes.
Example: When an index reaches 50GB or 30 days old, move it from Hot to Warm.

Cluster Architecture

Understanding how data is distributed is critical for preventing data loss.

[cite_start]

The Logic of Writes (Indexing) [cite: 567-569]

  1. Client sends data to Node 1 (Coordinating Node).
  2. Node 1 hashes the document ID to find which Shard it belongs to (e.g., Shard 0).
  3. Node 1 forwards data to the Primary Shard (on Node 3).
  4. Primary Shard writes data, then forwards it to Replica Shards (on Node 1 and 2) in parallel.
  5. Once Replicas confirm, Primary reports success.
[cite_start]Split Brain Warning: The PDF advises using an odd number of Master Nodes (e.g., 3) to avoid "Split Brain" scenarios where the cluster divides into two arguing halves[cite: 836].

Troubleshooting Guide

Status Codes

  • GREEN: All primary and replica shards are active.
  • YELLOW: All primary shards are active, but some replicas are missing. (Common in single-node dev setups).
  • RED: Some primary shards are missing. Data is missing. Panic.

Common Issues & Fixes

Symptom Probable Cause Fix
Status is Yellow Not enough nodes to hold replicas. Add another node or set number_of_replicas: 0 for dev.
"Split Brain" Network partition between masters. Ensure cluster.initial_master_nodes is set correctly and you have 3 masters.
Slow Search Too many shards for the hardware. Reindex into fewer shards (Shrink API).
