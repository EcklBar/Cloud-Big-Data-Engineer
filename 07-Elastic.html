Index Lifecycle Management (ILM)

Data has gravity. You cannot store petabytes on expensive NVMe drives forever. [cite_start]We use a Tiered Architecture [cite: 416-433].

Tier Function Hardware Spec
🔥 HOT Ingest & Recent Search High CPU, NVMe SSD
🍵 WARM Read-Only, 7-30 Days Old Standard SSD, Med CPU
❄️ COLD Compliance, >30 Days Cheap HDD / Object Store
Strategy: Use Kibana to automate this. "If index > 50GB or > 7 days old, move to Warm."

The Analyzer Pipeline

Before text is searchable, it must be analyzed. [cite_start]This pipeline transforms raw text into tokens [cite: 941-951].


Raw Text ➔ Char Filter ➔ Tokenizer ➔ Token Filter ➔ Inverted Index

Example: "I'm in the mood for drinking!"

  1. Char Filter: Strips HTML tags.
  2. Tokenizer: Splits on spaces. [I'm, in, the, mood, for, drinking]
  3. Token Filter: Lowercase & Stemming. [i'm, mood, drink]
[cite_start]

Test it in Console [cite: 981]

POST _analyze
{
  "tokenizer": "standard",
  "text": "I'm in the mood for drinking semi-dry red wine!"
}

Cluster Architecture

Elasticsearch is distributed by default. Understanding Shards is key to scaling.

[cite_start]

The Life of a Write Operation [cite: 567-569]

  1. Client hits a Coordinating Node.
  2. Node hashes the ID to find the Primary Shard.
  3. Data is written to Primary.
  4. Primary forwards data to Replica Shards in parallel.
  5. Once Replicas ack, the client gets a 200 OK.
[cite_start]Split Brain Warning: Always configure an odd number of Master-eligible nodes (e.g., 3) to prevent the cluster from splitting into two isolated islands[cite: 836].

Troubleshooting

Deciphering the traffic lights of Elastic.

Status Meaning Action Required
● GREEN Healthy None. All primary & replica shards are active.
● YELLOW Risk All Primary shards active, but Replicas are missing. Common in single-node dev setups.
● RED Critical Primary shards are missing. Data is unavailable. Check disk space or node connectivity immediately.
