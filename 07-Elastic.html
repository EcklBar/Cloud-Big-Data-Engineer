What is Elasticsearch?

Elasticsearch is a distributed, free and open search and analytics engine for all types of data, including textual, numerical, geospatial, structured, and unstructured.

Why is it so fast? It is built on top of Apache Lucene and uses a structure called an Inverted Index, which is designed for very fast full-text searches.
[cite_start]

Key Features [cite: 59, 170]

  • Distributed: Scales horizontally to handle petabytes of data.
  • RESTful API: Accessed via JSON over HTTP.
  • Near Real-Time (NRT): Slight latency (usually 1 second) from the time you index a document until it becomes searchable.

The ELK Stack

[cite_start]

"ELK" is the acronym for three open source projects: Elasticsearch, Logstash, and Kibana[cite: 67].

1. Beats (Collection)

Metaphor: Think of Beats as the "Delivery Trucks" picking up packages.
[cite_start]

Beats are lightweight data shippers that you install as agents on your servers to capture all sorts of operational data (think of them as "Agents")[cite: 98].

[cite_start]

They are individual packages written in Go, focused on single types of data (e.g., Filebeat for logs, Metricbeat for metrics)[cite: 98].

2. Logstash (Ingest)

Metaphor: Think of Logstash as the "Sorting & Packaging Center".
[cite_start]

Logstash is a server-side data processing pipeline that ingests data from a multitude of sources, transforms it, and then sends it to your "stash" (Elasticsearch)[cite: 85].

[cite_start]

It can derive structure from unstructured data using grok filters, anonymize sensitive fields, and aggregate data[cite: 87].

3. Elasticsearch (Store)

Metaphor: Think of Elasticsearch as the "Warehouse" with an ultra-fast filing system.

The heart of the stack. It is a distributed, RESTful search and analytics engine. [cite_start]It stores data as JSON documents and uses an inverted index to allow for near real-time search[cite: 59].

[cite_start]

Data is split into shards (horizontal partitions) which are distributed across nodes for scalability and high availability[cite: 474].

4. Kibana (Visualize)

Metaphor: Think of Kibana as the "Dashboard" in the control room.

Kibana is the visualization layer. [cite_start]It works on top of Elasticsearch to let users visualize data through charts, graphs, maps, and dashboards[cite: 74].

[cite_start]

It is the window into the Elastic Stack, allowing you to search, view, and interact with your data stored in Elasticsearch indices[cite: 76].

Core Terminology

Cluster & Node

A Cluster is a collection of one or more servers (nodes) that together hold your entire data. [cite_start]A Node is a single server that is part of your cluster[cite: 337].

Index

An index is a collection of documents that have somewhat similar characteristics. [cite_start]It is identified by a name (all lowercase) and is similar to a Database in RDBMS[cite: 189].

Document

A document is a basic unit of information that can be indexed. It is expressed in JSON format. [cite_start]Similar to a Row in a table[cite: 183].

[cite_start]Analogy[cite: 204]:
Relational DB ⇒ Databases ⇒ Tables ⇒ Rows ⇒ Columns
Elasticsearch ⇒ Indices ⇒ Types (Deprecated) ⇒ Documents ⇒ Fields

Architecture: Sharding & Replication

An index can store a large amount of data that can exceed the hardware limits of a single node. [cite_start]To solve this, Elasticsearch allows you to subdivide your index into multiple pieces called shards[cite: 474].

Shards

    [cite_start]
  • Allows you to horizontally split/scale your content volume[cite: 475].
    • [cite_start]
  • Allows you to distribute and parallelize operations across shards (performance)[cite: 475].

Replicas

[cite_start]

To ensure high availability, Elasticsearch allows you to make one or more copies of your index’s shards into replica shards[cite: 508].

    [cite_start]
  • Provides high availability in case a node/shard fails[cite: 509].
    • [cite_start]
  • Allows you to scale out your search volume/throughput (searches can be executed on replicas in parallel)[cite: 510].

The Inverted Index

This is the magic behind the speed. [cite_start]Instead of storing strings directly, Elasticsearch lists every unique word that appears in any document and identifies all of the documents each word occurs in[cite: 229].

Elasticsearch REST API

Interact with your cluster using simple JSON over HTTP.

GET/{index}/_doc/{id}

Retrieve a document.

GET /customer/_doc/1
PUT/{index}/_doc/{id}

Index/Update document.

PUT /customer/_doc/1 { "name": "John" }
POST/{index}/_search

Search query.

POST /customer/_search { "query": { "match_all": {} } }
DELETE/{index}

Delete index.

DELETE /customer
