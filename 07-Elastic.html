1. Data Collection (Beats)

Beats are lightweight data shippers that you install as agents on your servers.
They collect data (like logs from Filebeat or metrics from Metricbeat) and send it to Logstash or directly to Elasticsearch.
Metaphor: Think of Beats as the "Delivery Trucks" picking up packages.

Filebeat Simulation: Enter a raw log message to begin the pipeline.

2. Data Transformation (Logstash)

Logstash is a server-side data processing pipeline. It ingests data from multiple sources, transforms it (e.g., parsing unstructured text into JSON using Grok filters), and sends it to a "stash" like Elasticsearch.
Metaphor: Think of Logstash as the "Sorting & Packaging Center".

3. Storage & Processing (Elasticsearch)

Elasticsearch is the heart of the stack. It is a distributed search and analytics engine. Data is stored in Shards (Primary and Replica) distributed across Nodes (Servers). It uses an Inverted Index to make searching incredibly fast.
Metaphor: Think of Elasticsearch as the "Warehouse" with an ultra-fast filing system.

4. Visualization (Kibana)

Kibana is the window into your data. It sits on top of Elasticsearch and lets you visualize your data with charts, graphs, maps, and dashboards. It is used for analytics and application monitoring.
Metaphor: Think of Kibana as the "Dashboard" in the control room.
