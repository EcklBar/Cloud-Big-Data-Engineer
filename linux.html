🐧 Linux Crash Course

Beginner to Intermediate

1. Linux Overview & Philosophy

Linux is an open-source operating system modeled on UNIX. It powers everything from supercomputers and web servers to Android phones and smart appliances.

The Philosophy

  • Everything is a file: Hardware, processes, and data are all represented as files.
  • Small, single-purpose tools: Programs should do one thing and do it well.
  • Chaining: Programs are designed to work together via text streams.

The Kernel vs. Shell

The Kernel is the core that talks to hardware. The Shell is the command-line interface (CLI) you use to talk to the kernel.

2. File System Structure

The Linux file system is a single hierarchical tree, starting at the root directory /.

Key Directories

/       # Root directory (start of everything)
/home   # User personal files (e.g., /home/alice)
/bin    # Essential user binaries (commands like ls, cp)
/etc    # System configuration files
/var    # Variable files (logs, spools)
/root   # Home directory for the Superuser (root)

3. Basic Commands

Mastering navigation and file manipulation is the first step.

Navigation

pwd             # Print Working Directory (where am I?)
ls -la          # List files (long format, show hidden)
cd /var/log     # Change Directory
cd ..           # Go up one level
cd ~            # Go to your home directory

File Operations

touch file.txt      # Create an empty file
mkdir myfolder      # Make a directory
cp file.txt copy.txt# Copy a file
mv file.txt new.txt # Move (or rename) a file
rm file.txt         # Remove (delete) a file
cat file.txt        # Display file content
📝 Challenge: Cleanup

How would you delete a folder named "temp_data" and everything inside it?

rm -r temp_data (The -r flag stands for recursive)

4. Advanced Commands

The real power of Linux comes from filtering text and chaining commands.

Grep and Find

grep "error" log.txt    # Find lines containing "error" in log.txt
find . -name "*.jpg"    # Find all .jpg files in current folder

Pipes (|) and Redirection (>)

Pipes take the output of one command and make it the input of another.

ls | grep "data"        # List files, but only show ones with "data" in name
cat list.txt | sort     # Read file and sort the lines alphabetically

Redirection saves output to a file.

echo "Log entry" > log.txt  # Overwrite file with text
echo "Next entry" >> log.txt # Append text to end of file
📝 Challenge: Counting

How would you count how many files in the current directory contain the word "conf"?

ls | grep "conf" | wc -l (ls lists files, grep filters them, wc -l counts lines)

5. Permissions & Ownership

Linux is multi-user. Every file has an Owner, a Group, and permissions for Everyone else.

Understanding rwx

Permissions appear as drwxr-xr--.

  • r: Read
  • w: Write
  • x: Execute (run as program or enter directory)

Changing Permissions

chmod 755 script.sh     # Owner:rwx, Group:rx, Others:rx
chmod +x script.sh      # Make file executable for everyone
chown user:group file   # Change ownership
📝 Challenge: Privacy

How do you make a file "secret.txt" readable and writable ONLY by the owner, with no access for anyone else?

chmod 600 secret.txt (6=rw-, 0=---, 0=---)

6. Processes & Monitoring

Every running program is a process with a Process ID (PID).

Monitoring

top       # Real-time view of processes (CPU/RAM usage)
ps aux    # Snapshot of all running processes
df -h     # Disk usage (human readable)
free -m   # Memory usage

Managing

kill 1234       # Ask process ID 1234 to stop
kill -9 1234    # Force kill process ID 1234
📝 Challenge: The Zombie

You have a stuck program named "firefox". You don't know the PID. How can you kill it by name?

pkill firefox or killall firefox

7. Networking Basics

Linux is the backbone of the internet. Here are tools to check connectivity.

ping google.com       # Check if you can reach a host
ip addr show          # Show IP addresses (replaces ifconfig)
curl google.com       # Download/View a webpage in terminal
ssh user@host         # Secure Shell: log into remote machine
📝 Challenge: Download

How do you download a file from "http://example.com/image.png" and save it as "pic.png"?

curl -o pic.png http://example.com/image.png or wget -O pic.png ...

8. Package Management

Linux uses repositories to install software, not random .exe downloads.

Debian/Ubuntu (apt)

sudo apt update        # Refresh software list
sudo apt install git   # Install Git
sudo apt remove git    # Uninstall Git

RedHat/CentOS (yum/dnf)

sudo yum install git   # Install Git
📝 Challenge: Upgrade

What command updates all currently installed packages to their latest versions on Ubuntu?

sudo apt upgrade

9. Shell Scripting

Automate boring tasks by putting commands in a file.

Create a file named hello.sh:

#!/bin/bash
# This is a variable
NAME="User"

# Logic
if [ -d "/tmp" ]; then
    echo "Hello $NAME, /tmp directory exists!"
else
    echo "Something is wrong."
fi

Run it with ./hello.sh (remember to chmod +x it first!).

📝 Challenge: Loop

Write a one-liner loop to echo numbers 1 through 5.

for i in {1..5}; do echo $i; done

10. Troubleshooting

When things break, Linux gives you clues.

  • Logs: Check /var/log/syslog or /var/log/messages.
  • dmesg: Prints kernel ring buffer messages (hardware issues).
  • man: The manual. Run man [command] to learn how to use it.
📝 Challenge: Help

You forgot how the `tar` command works. How do you find the manual inside the terminal?

man tar
