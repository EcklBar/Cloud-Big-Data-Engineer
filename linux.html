🚀 Introduction

Understanding the Linux ecosystem[cite: 7].

Why Linux?

  • Ubiquitous: Powers 97% of supercomputers and the majority of cloud servers[cite: 17, 19].
  • Core Concept: "Everything is a file" - Even hardware and processes are represented as files[cite: 359].
  • Distributions: Red Hat (Enterprise), CentOS (Free Enterprise), Ubuntu (Popular), Debian (Stable)[cite: 38, 51, 55].

⚠️ Crucial Warning: Case Sensitivity

Linux files are Case Sensitive[cite: 352, 353].

📁 Directories

Navigation and directory structure management.

pwd
Print Working Directory.
pwd # Show full path
ls
List content.
ls # Basic list
ls -l # Long format (details)
ls -la # Include hidden files (dot files) [cite: 266]
ls -lh # Human readable sizes
cd
Change Directory.
cd /path/to/dir # Absolute path [cite: 236]
cd .. # Go up one level (Parent) [cite: 223]
cd ~ # Go to home dir [cite: 209]
cd - # Go to previous dir [cite: 209]
mkdir
Create directories.
mkdir new_folder # Must provide name [cite: 295]
mkdir -p dir1/sub1/sub2 # Recursive path [cite: 308]
rmdir
Remove empty directories only[cite: 319].
rmdir empty_folder
rmdir -p path/to/empty
tree
Visualize directory tree.
tree -L 2 # Max depth 2

📄 File Operations

Create, copy, move, delete, and find files.

touch
Create file / Update timestamp.
touch file.txt # Creates empty file if missing [cite: 369]
touch -t 202312251200 file # Set specific time [cite: 370]
cp
Copy files/folders.
cp source dest
cp -r folder1 folder2 # Recursive (for dirs) [cite: 404]
cp file1 file2 dir/ # Copy multiple files to dir [cite: 404]
mv
Move or Rename.
mv old.txt new.txt # Rename file [cite: 413]
mv dir1 dir2 # Rename/Move directory [cite: 417]
rm
Remove (Delete). No Recycle Bin! [cite: 381]
rm file.txt
rm -i file.txt # Interactive (ask)
rm -rf folder/ # Force recursive delete [cite: 384]
find
Advanced search[cite: 831].
find . -name "*.py"
find / -size +100M
find . -mtime -7 # Last 7 days
locate
Quick DB search.
locate filename
updatedb # Update DB (root)
ln
Links.
ln -s target link # Symbolic
ln file hardlink # Hard
file
Check file type[cite: 345].
file doc.pdf
file *

📝 Content Viewing

Reading and inspecting file contents.

cat
Print content / Concatenate[cite: 460].
cat file.txt
cat f1 f2 > merged # Join files [cite: 465]
cat > file.txt # Create file (Ctrl+C to stop)
head
Top of file.
head -5 file.txt # First 5 lines
head -c 100 file # First 100 bytes
tail
Bottom of file.
tail -n 20 file.txt # Last 20 lines [cite: 448]
tail -c 100 file # Last 100 bytes [cite: 449]
tail -f log.txt # Live monitor
tac
Reverse cat (Bottom-up).
tac file.txt
less / more
Paged view[cite: 494].
less file.txt # Scrollable
more file.txt # Forward only
grep
Search text[cite: 753].
grep "pattern" file
grep -i "pat" file # No-case
grep -v "pat" file # Invert match
ripgrep (rg)
Modern, faster grep.
rg "pattern"
rg -t py "import" # Py files only
diff
Compare files.
diff file1 file2
diff -y file1 file2 # Side-by-side

🔐 Permissions

User, Group, Others access control[cite: 981].

📂 Directory vs File Permissions

  • R (Read): File: View content | Dir: List files (ls)
  • W (Write): File: Edit content | Dir: Create/Delete files (touch/rm)
  • X (Execute): File: Run script | Dir: Enter directory (cd)
ls -l
View permissions.
ls -l file
chmod
Change mode[cite: 993].
chmod +x script
chmod 755 file # rwx-rx-rx
chmod 600 key # rw-------
chown
Change Owner[cite: 963].
chown user:group file
chown -R user dir/
