📚 הגדרות Docker

Docker Image
תבנית לקריאה בלבד (read-only) המכילה את כל הקבצים, הספריות והתלויות הנדרשות להרצת אפליקציה בקונטיינר Docker.
Docker Container
מופע פועל (running instance) של image. כלומר, חבילה הניתנת להרצה הכוללת את הקוד, הספריות והתלויות הנדרשות להרצת האפליקציה.
Dockerfile
סקריפט המכיל הוראות ליצירת image. קובץ טקסט עם פקודות המגדירות כיצד לבנות את ה-image.
Docker Registry
מאגר לאחסון, שיתוף וניהול Docker images. כולל Docker Hub, Amazon ECR, Azure Container Registry ו-Google Cloud Container Registry.
Docker Engine
אפליקציה לניהול Docker containers. כוללת שרת (Docker daemon), כלי שורת פקודה (Docker client) ו-API לאינטראקציה עם ה-daemon.
Docker Client
כלי שורת פקודה לאינטראקציה עם Docker Engine לניהול images וקונטיינרים.
Docker Daemon
תהליך רקע (background process) המנהל Docker images וקונטיינרים בהתאם לפקודות שנשלחות מה-Docker client.
Docker Compose
כלי להגדרה והרצה של אפליקציות Docker מרובות קונטיינרים באמצעות קובץ YAML לתצורת השירותים.

❓ קבלת עזרה

הצג גרסת Docker
docker --version
הצג מידע על מערכת Docker
docker info
קבל עזרה על Docker
docker --help
קבל עזרה על פקודת Docker ספציפית
docker [command] --help

🚀 הרצת קונטיינרים

הרץ קונטיינר
docker run [image]
הרץ קונטיינר בדיקה לוודא שההתקנה עובדת
docker run hello-world
הרץ קונטיינר ולאחר מכן השתמש בו להרצת פקודה
docker run [image] [command]
הרץ Python באופן אינטראקטיבי
docker run python -c "print('Python is Docker')"
הרץ R והדפס מודל
docker run rocker/r-base r -e "print(lm(dist~speed, cars))"
הרץ קונטיינר במצב אינטראקטיבי
docker run --interactive --tty rocker/r-base
הרץ קונטיינר והסר אותו כשסיימת
docker run --rm my_mysql
הרץ image ברקע (detached mode)
docker run --detach postgres
הרץ image עם שם מוגדר
docker run --name [name] redis
הרץ image כמשתמש ספציפי
docker run --user doctordocker mongo

🔍 בדיקת קונטיינרים

הצג את כל הקונטיינרים הפועלים
docker ps
הצג את כל הקונטיינרים (כולל לא פעילים)
docker ps --all
סנן קונטיינר פועל לפי שם
docker ps -f "name=[container-name]"
הצג לוגים קיימים של קונטיינר
docker logs [container-id]
הצג לוגים חיים של קונטיינר (מעקב בזמן אמת)
docker logs -f [container-id]
הצג לוג עם חותמות זמן
docker logs --timestamps --follow [container-id]

⚙️ ניהול קונטיינרים

צור קונטיינר מ-image
docker create [image]
הפעל קונטיינר
docker start [container]
צור קונטיינר עם shell אינטראקטיבי
docker create --name py --interactive --tty python
צור image חדש מקונטיינר
docker container commit [container]
עצור קונטיינר
docker stop [container]
השהה קונטיינר (pause)
docker pause [container]
בטל השהיה (unpause)
docker unpause [container]
הרוג קונטיינר (kill)
docker kill [container]
הרוג והסר קונטיינר בפקודה אחת
docker rm --force [container]
הסר קונטיינר שהופסק
docker rm [container]
הסר קונטיינר (תחביר מודרני)
docker container rm [container]
הסרה בכוח (תחביר מודרני)
docker container rm --force [container]
עצור והסר בפקודה אחת
docker rm -f [container]
עצור ואז הפעל מחדש קונטיינר
docker restart [container]
מחק את כל הקונטיינרים שהופסקו
docker container prune

🔨 בניית Images

בנה image
docker build [path]
בנה image מהתיקייה הנוכחית
docker build .
בחר שם בעת בניית image
docker build -t first_image .
בנה image עם tag
docker build --tag [name:tag] [path]
בנה image עם גרסה ספציפית
docker build --tag myimage:2023-edition .
בנה image ללא שימוש ב-cache
docker build --no-cache [path]

משוך image

תן שם חדש ל-image
docker tag <old-name> <new-name>
שמור image לקובץ
docker save -o <file-name> <image-name>
טען image מקובץ
docker load -i <file-name>
ראה את ההוראות שנוצרו ליצירת image
docker history <image-name>
הצג שימוש בדיסק של Docker
docker system df
הסר image (תחביר קצר)

רשימת images (תחביר מודרני)
docker image ls
הסר images שאינם בשימוש
docker image prune
הסר build cache
docker builder prune

📝 הוראות Dockerfile

התחל Dockerfile מ-image
FROM <image-name>
הגדר משתנה סביבה לבנייה ולהרצה
ENV [name]=[value]
הגדר משתנה זמן בנייה
ARG [name]=[default_value]
הגדר את תיקיית העבודה
WORKDIR [path]
עבור למשתמש
USER [username]
העתק קובץ מקומי ל-image
COPY [existing path] [image path]
הוסף פקודת shell ל-image
RUN <valid-shell-command>
התקנה ללא אינטראקציה
RUN apt-get install -y python3
הרץ פקודה בעת השקת הקונטיינר
CMD ["executable", "param1"]
העתק קבצים מהמארח ל-image
COPY <src-path-on-host> <dest-path-on-image>
העתק תיקייה
COPY <src-folder> <dest-folder>
הגדר פקודת ברירת מחדל
CMD <shell-command>
שנה משתמש נוכחי
USER <user-name>
צור משתנה זמן בנייה בלבד
ARG <name>=<value>
צור משתנה סביבה
ENV <name>=<value>
בנה image מ-Dockerfile
docker build /location/to/Dockerfile
בנה מהתיקייה הנוכחית
docker build .
דרוס ARG בזמן build
docker build --build-arg <name>=<value> .
חשוף פורט
EXPOSE <port>
הגדר נקודת כניסה
ENTRYPOINT ["executable", "param1"]
הוסף metadata ל-image
LABEL key="value"
צור נקודת עיגון
VOLUME ["/data"]
הוסף קבצים וחלץ ארכיונים
ADD <src> <dest>
הגדר בדיקת בריאות
HEALTHCHECK --interval=30s CMD curl -f http://localhost/ || exit 1

✨ שיטות עבודה מומלצות

שמור על images קטנים - הורד, חלץ ונקה בהוראת RUN אחת
RUN curl <file_url> -o <dest> \ && unzip <dest>/<file>.zip -d <dir> \ && rm <dest>/<file>.zip
השתמש ב-multi-stage builds
FROM node:14 AS build WORKDIR /app COPY . . RUN npm install && npm run build FROM nginx:alpine COPY --from=build /app/dist /usr/share/nginx/html
השתמש ב-.dockerignore
# .dockerignore node_modules .git *.log .env
הפעל כמשתמש לא-root
RUN adduser -D myuser USER myuser
צא מתצוגת לוגים חיים
CTRL+C

💡 טיפים חשובים:

  • השתמש ב-images רשמיים כבסיס
  • מזער את מספר השכבות (layers)
  • סדר את ההוראות מהכי פחות משתנות לכי יותר משתנות
  • אל תשמור סודות ב-images
  • השתמש ב-health checks

🌐 רשתות Docker

הצג את כל הרשתות
docker network ls
צור רשת חדשה
docker network create <network-name>
חבר קונטיינר לרשת
docker network connect <network-name> <container-name>
נתק קונטיינר מרשת
docker network disconnect <network-name> <container-name>
בדוק רשת
docker network inspect <network-name>
הסר רשת
docker network rm <network-name>
הסר רשתות שאינן בשימוש
docker network prune
הרץ קונטיינר עם רשת ספציפית
docker run --network=<network-name> <image>
פרסם פורט קונטיינר למארח
docker run -p <host-port>:<container-port> <image>
פרסם את כל הפורטים החשופים
docker run -P <image>

🔧 פתרון בעיות ופקודות מתקדמות

הרץ פקודה בקונטיינר פועל
docker exec -it <container> <command>
פתח bash shell בקונטיינר פועל
docker exec -it <container> /bin/bash
העתק קבצים מקונטיינר למארח
docker cp <container>:<src-path> <dest-path>
העתק קבצים ממארח לקונטיינר
docker cp <src-path> <container>:<dest-path>
בדוק פרטי קונטיינר
docker inspect <container>
הצג שימוש במשאבים
docker stats
הצג שימוש במשאבים לקונטיינר ספציפי
docker stats <container>
הצג תהליכים בקונטיינר
docker top <container>
הצג מיפוי פורטים
docker port <container>
הצג שינויים בקונטיינר
docker diff <container>
המתן ליציאת קונטיינר והצג קוד יציאה
docker wait <container>
עדכן תצורת קונטיינר
docker update --restart=unless-stopped <container>
נקה הכל (קונטיינרים, images, רשתות, volumes)
docker system prune -a --volumes
ייצא קונטיינר כארכיון tar
docker export <container> > <file>.tar
ייבא ארכיון tar כ-image
docker import <file>.tar <image-name>
התחבר לקונטיינר פועל
docker attach <container>
הרץ קונטיינר עם הגבלות משאבים
docker run --memory=512m --cpus=1.5 <image>
הגדר מדיניות הפעלה מחדש
docker run --restart=always <image>
