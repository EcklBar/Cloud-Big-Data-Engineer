üèÉ Jobs and Runners
üíº Jobs
A set of steps in a workflow. Each job is independent and can run in parallel, but can be configured to depend on other jobs.
üñ•Ô∏è Runners
The compute machines where all steps in a single job are executed. Available runners include Ubuntu Linux, Windows, and macOS.

‚öôÔ∏è Pull Request Workflow Configuration
üîÑ Event Trigger Change
Modify your existing workflow to trigger on pull requests instead of pushes. This enables automated testing before code merges.

‚öñÔ∏è Push vs Pull Request Triggers

üì§ Push Trigger

Runs after code is merged

on: push

üîÑ PR Trigger

Runs before code is merged

on: pull_request

üéØ Key Difference

Pull Request triggers: Test changes BEFORE merging
Push triggers: Test changes AFTER merging

# Before: Push trigger name: CI on: push: branches: [ main ] # After: Pull Request trigger name: CI on: pull_request: branches: [ main ] # Target branch (where PR merges into)
üéØ Target Branch Specification
The "branches" key specifies the target branch of the pull request - where the code will be merged into (typically "main").
