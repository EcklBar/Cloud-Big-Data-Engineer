🔍 What is GitHub Actions?
00:05 - 00:37
GitHub Actions (GHA) Definition
GitHub Actions is a built-in automation and CI/CD system provided by GitHub that allows you to automate your build, test, and deployment pipeline directly within your GitHub repositories.

💡 Key Concept: Pipeline

A pipeline is a sequence of interconnected steps or stages that represent the flow of work and data in a software development or deployment process.

Pipeline Flow: Code → Build → Test → Deploy → Monitor

🔧 Steps and Actions
02:30 - 03:27
👣 Steps
Individual units of work, executed in order and dependent on the previous step. Each step runs on the same compute machine and can share data.
⚡ Actions
Custom applications that combine complex but frequently repeated tasks specific to the GitHub Actions platform.

📝 Example Step Sequence:

1️⃣ Build Application (Compile Code)
2️⃣ Test Application (Run Tests)
3️⃣ Deploy Application (Shell Script)

🎯 Popular Actions Examples:

• Checkout Repository • Comment on Pull Request • Setup Node.js • Deploy to AWS

🏃 Jobs and Runners
03:27 - 03:55
💼 Jobs
A set of steps in a workflow. Each job is independent and can run in parallel, but can be configured to depend on other jobs.
🖥️ Runners
The compute machines where all steps in a single job are executed. Available runners include Ubuntu Linux, Windows, and macOS.

📄 Multiline Strings: Block Scalar Format

🎯 Block Scalar Strings
A way to represent multi-line strings with line breaks and indentation while preserving their formatting. Perfect for code snippets, shell commands, and configurations in CI/CD workflows.

🛠️ Two Styles + Three Chomping Indicators

Styles: Literal (|) & Fold (>)
Chomping: Clip (default), Strip (-), Keep (+)

🔄 Fold Style (>)
01:20 - 02:05
📦 Fold Style Collapses Lines
Removes line breaks and converts them to spaces. Useful for long text that spans multiple lines for readability.

⚖️ Literal vs Fold Comparison

📝 Same Text, Different Results:

Literal (|): 
Line 1
Line 2
Line 3
Fold (>): 
Line 1 Line 2 Line 3
message: > This is a very long message that spans multiple lines for better readability in your YAML file but will be folded into a single line when parsed. # Result: "This is a very long message that spans multiple lines for better readability in your YAML file but will be folded into a single line when parsed."
✂️ Chomping Indicators
02:05 - 03:12
🎛️ Control Final Newlines
Special instructions that tell YAML how to handle blank lines at the end of multi-line text.

📎 Clip

Default (adds one newline)

✂️ Strip (-)

Removes all newlines

📌 Keep (+)

Preserves all newlines

🔄 Dynamic Value Injection
03:12 - 04:18
💉 Expressions & Variables
Placeholders that represent values that can change when YAML is processed. Essential for flexible GitHub Actions workflows!
# GitHub Actions Expression Examples: name: CI Pipeline on: push jobs: build: runs-on: ${{ matrix.os }} strategy: matrix: os: [ubuntu-latest, windows-latest] env: DATABASE_URL: ${{ secrets.DATABASE_URL }} BRANCH_NAME: ${{ github.ref_name }} steps: - name: Checkout run: echo "Building on ${{ runner.os }}"

🎮 Try Dynamic Expressions

${{ secrets.API_KEY }} - Secure variables
${{ github.actor }} - GitHub context
${{ env.NODE_VERSION }} - Environment variables

📚 Multi-Document YAML
04:18 - 05:19
📑 Multiple Documents in One File
Store multiple independent YAML documents in a single file, separated by three dashes (---). Great for configuration management!
