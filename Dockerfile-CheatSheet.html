🧾 Dockerfile Cheat Sheet

A single-page reference for building efficient container images.

🚀 1. Basic Structure

The fundamental skeleton of most Python-based Dockerfiles.

FROM <base-image>

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

CMD ["python", "main.py"]

🧱 2. Base Image Rules

  • Use official images when possible.
  • Examples: python:3.10 node:20 apache/airflow:2.9.2
  • Avoid reinstalling software already included in the base image.

⚙️ 3. ARG vs ENV

ARG (Build Time) Only exists during the docker build process. 
ARG PY_VER=3.10
FROM python:${PY_VER}
ENV (Runtime) Persists inside the running container. 
ENV AIRFLOW_HOME=/opt/airflow

📁 4. COPY Order (Caching)

Order matters for Docker layer caching. Copy dependencies before code.

# 1. Copy requirements first
COPY requirements.txt .
# 2. Install dependencies (Cached unless reqs change)
RUN pip install --no-cache-dir -r requirements.txt
# 3. Copy source code (Changes often)
COPY . .

💾 5. Always Use .dockerignore

Create a .dockerignore file to avoid copying unnecessary files:

__pycache__/
.env
*.log
.git

This keeps images small and builds fast.

🧩 6. Installing Dependencies

Python Packages:

RUN pip install --no-cache-dir -r requirements.txt

OS Packages (e.g., Airflow):

USER root
RUN apt-get update && apt-get install -y nano
USER airflow

🧩 7. Multi-Stage Builds

Optional but powerful. Makes final images tiny by discarding build tools.

# Stage 1: Builder
FROM python:3.10 as builder
RUN pip install --prefix=/install -r requirements.txt

# Stage 2: Final Image
FROM python:3.10
COPY --from=builder /install /usr/local

💡 8. Keep Images Small

  • ✅ Use --no-cache-dir in pip.
  • ✅ Use .dockerignore religiously.
  • ✅ Use slim base images (e.g., python:3.10-slim).
  • ✅ Avoid installing unnecessary debugging tools (vim, curl) unless needed.

🏃 9. CMD vs ENTRYPOINT

CMD (Default Command) Can be overridden by arguments passed to docker run. 
CMD ["python", "main.py"]
ENTRYPOINT (Fixed Executable) Difficult to override. Often used for wrappers like dumb-init. 
ENTRYPOINT ["/usr/bin/dumb-init", "--"]
CMD ["airflow", "scheduler"]

🔐 10. User Best Practices

Don't run everything as root for security reasons.

If you switch to root to install OS packages, always switch back:

USER airflow

🎯 Quick Memory Rules (Super Easy)

  • 1. Start with a base image
  • 2. Set WORKDIR
  • 3. Copy requirements → install → then copy code
  • 4. Use .dockerignore
  • 5. Use CMD to run app
  • 6. NEVER reinstall Airflow inside the official image
