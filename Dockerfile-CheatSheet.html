🧾 Dockerfile Cheat Sheet

🎓 Basics 🚀 Production
Based on "Dockerfile Basics" Guide

🚀 1. Basic Structure PRO

The standard skeleton for most applications.

FROM python:3.10
WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt
COPY . .
CMD ["python", "main.py"]
💡 Production Upgrades:
  • Pinned version 3.10.4-slim instead of generic tag.
  • Non-root user for security.
  • Environment variables to stop Python buffering.
FROM python:3.10.4-slim
ENV PYTHONUNBUFFERED=1
WORKDIR /app
RUN useradd -m appuser
...
USER appuser

🧱 2. Base Image Rules

Use official images when possible.

  • python:3.10
  • node:20
⚠️ Pro Tip:

Avoid latest. Always use specific versions (e.g., node:20.1-alpine) to prevent your build from breaking randomly when a new version is released.

⚙️ 3. ARG vs ENV

ARG: Build time only.
ENV: Runtime (persists in container).

🔒 Security Warning:

NEVER use ENV for passwords or API keys. They are visible in docker history. Use BuildKit secrets instead.

📁 4. COPY Order

Copy dependencies before code to use caching.

COPY requirements.txt .
RUN pip install ...
COPY . .

💾 5. Use .dockerignore

Exclude these to keep images small:

__pycache__/
.git
.env
*.log

🧩 6. Installing Dependencies

Standard installation:

RUN apt-get update && apt-get install -y nano
🧹 Clean up cache!

Combine commands and remove lists to save space.

RUN apt-get update && apt-get install -y \
    nano \
    && rm -rf /var/lib/apt/lists/*

🧩 7. Multi-Stage Builds

Build in one image, copy only artifacts to a smaller one.

FROM golang:1.18 as builder
RUN go build -o app .

FROM alpine:latest
COPY --from=builder /app /app

🏃 9. CMD vs ENTRYPOINT

CMD: Default arguments.
ENTRYPOINT: Fixed executable.

🛑 Signal Handling:

Always use JSON syntax ["cmd"]. The shell form CMD python app.py breaks Ctrl+C and graceful shutdowns.

🔐 10. User Best Practices

Don't run as root.

USER airflow

If your base image doesn't have a user, create one:

RUN adduser -D myuser
USER myuser
