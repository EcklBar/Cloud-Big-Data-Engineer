מבוא ל-Git

מה זה בקרת גרסאות?

  • מאחסנת ומנהלת כל גרסה ושינוי בקוד
  • שיתוף פעולה על קוד מקור וניהול גרסאות שחרור
  • חזרה לגרסאות קודמות כאשר מתגלים באגים
  • VSC = מערכת בקרת גרסאות

אודות Git

  • נוצר על ידי לינוס טורוואלדס ב-2005 (עבור הליבה של לינוקס)
  • מחליף כלים ישנים (CVS, SVN)
  • מתמקד בהפצת קוד מקור (לכל אחד יש עותק מלא)
  • מיישם אחסון קבצים יעיל במיוחד
🚀 שירותי VCS

GitHub

  • שירות אירוח מבוסס-ווב לקוד
  • בקרת גישה, ניהול משימות, אוטומציה (Actions)
  • הסטנדרט בתעשייה

אלטרנטיבות

  • GitLab (חזק ב-CI/CD)
  • BitBucket (אינטגרציה עם Jira)

יסודות Git

מושגי יסוד

  • Repository (Repo): המאגר בו נשמר הפרויקט
  • Commit: שמירת נקודת ציון (Snapshot)
  • Branch: ענף פיתוח נפרד

טיפ למהנדסי נתונים: .gitignore

  • חשוב מאוד למנוע העלאת קבצי דאטה רגישים או גדולים
  • צור קובץ .gitignore בתיקייה הראשית
  • הוסף סיומות כגון *.csv, *.parquet, .env
git init
- אתחל מאגר חדש
git clone <url>
- שכפל מאגר קיים
git status
- בדוק סטטוס קבצים

זרימת עבודה (Workflow)

תיקיית עבודה
←
Staging (אינדקס)
←
Repository (מאגר)

שלושת השלבים

  • Working Dir: איפה שאתה עורך את הקוד
  • Staging Area: קבצים שמיועדים להיכנס ל-Commit הבא
  • Repository: ההיסטוריה השמורה
git add .
- הוסף את כל השינויים ל-Staging
git commit -m "msg"
- שמור את השינויים בהיסטוריה
git push
- דחף את השינויים לשרת

ענפים (Branching)

🌳 למה ענפים?

מאפשרים לעבוד על פיצ'רים חדשים או תיקוני באגים במקביל, מבלי לשבור את הקוד הראשי (Main/Master).

git checkout -b new-feature
- צור ענף חדש ועבור אליו
git checkout main
- חזור לענף הראשי
git merge new-feature
- מזג את השינויים מהענף לתוך הראשי

קונפליקטים (Conflicts)

מהו קונפליקט?

כששני אנשים משנים את אותה שורה באותו קובץ, Git לא יודע איזו גרסה נכונה. עליך להחליט ידנית.

איך פותרים?

  1. פתח את הקובץ הבעייתי
  2. חפש את הסימונים <<<<<<<
  3. בחר את הקוד הנכון ומחק את הסימונים
  4. בצע git add ו-git commit

פקודות שימושיות נוספות

Logs & History

git log --oneline --graph
- הצג עץ קומיטים ויזואלי
git diff
- ראה שינויים לפני הוספה
git blame filename
- ראה מי כתב כל שורה

Undoing

git restore filename
- בטל שינויים בקובץ (לפני add)
git reset HEAD~1
- בטל את ה-Commit האחרון אך שמור שינויים
git revert <hash>
- צור קומיט הפוך המבטל שינוי קודם

תרגול

🎯 משימה 1: אתחול
  1. צור תיקייה חדשה במחשב.
  2. פתח טרמינל והרצת git init.
  3. צור קובץ data.csv וקובץ .gitignore.
  4. הוסף את ה-csv ל-gitignore.
  5. נסה לעשות git add . וודא שה-csv לא מתווסף.
🎯 משימה 2: קונפליקט
  1. צור ענף חדש. שנה שורה בקובץ טקסט. בצע commit.
  2. חזור ל-main. שנה את אותה שורה אחרת. בצע commit.
  3. נסה למזג (merge) את הענף ל-main.
  4. פתור את הקונפליקט שנוצר.
