    🚀 Intro & Concepts

    Apache Kafka is a distributed streaming platform based on the publish-subscribe model. [cite_start]It is designed to be fast, scalable, and fault-tolerant[cite: 94].

    [cite_start]

    ETL vs. Data Streaming [cite: 35-71]

    Old World (ETL): Batch processing, slow, resource-intensive, runs on schedule (not real-time).

    New World (Streaming): Real-time processing, low latency, fault-tolerant, scalable, handles continuous event streams.

    Broker
    A Kafka server. [cite_start]A cluster is made of multiple brokers[cite: 529].
    Topic
    A logical channel for data. [cite_start]Producers write to topics, Consumers read from them[cite: 570].
    Partition
    Topics are split into partitions for scalability. [cite_start]Ordering is guaranteed only within a partition[cite: 606].
    Offset
    [cite_start]
    A unique incremental ID assigned to every message in a partition[cite: 598].
    Replication
    [cite_start]
    Copies of data across brokers for High Availability (HA)[cite: 647].
    Zookeeper
    Manages brokers, leader election, and cluster membership. [cite_start]Mandatory for Kafka[cite: 762].

    Consumer

    Consumer Group
    A group of consumers working together. [cite_start]Each partition is consumed by only one member of the group for load balancing [cite: 723-725].

    💻 CLI Commands

    Essential commands for managing your Kafka cluster. (Paths may vary based on installation).

    1. [cite_start]Service Management [cite: 925-927]

    Start Zookeeper
    bin/zookeeper-server-start.sh config/zookeeper.properties
    Start Kafka
    bin/kafka-server-start.sh config/server.properties

    2. [cite_start]Topic Management [cite: 964, 979]

    Create Topic
    Create a topic with specific replication and partitions.
    bin/kafka-topics.sh --create --zookeeper localhost:2181 --replication-factor 1 --partitions 1 --topic kafka-tst-01
    List Topics
    Show all available topics.
    bin/kafka-topics.sh --zookeeper localhost:2181 --list
    Describe Topic
    View partition and leader details.
    bin/kafka-topics.sh --zookeeper localhost:2181 --describe --topic kafka-tst-01

    3. [cite_start]Console Clients [cite: 992, 1003]

    Console Producer
    Send messages manually from CLI.
    bin/kafka-console-producer.sh --broker-list localhost:9092 --topic kafka-tst-01
    Console Consumer
    Read messages from CLI.
    bin/kafka-console-consumer.sh --bootstrap-server localhost:9092 --topic kafka-tst-01 --from-beginning

    🧩 Core APIs

    [cite_start]

    The 4 main APIs used for development[cite: 777].

    1. Producer API
    Allows applications to send data to Kafka topics.
    2. Consumer API
    Allows applications to read streams of data from topics.
    3. Streams API
    Allows transforming data (filter, map, aggregate) from input topics to output topics.
    4. Connect API
    Reusable source/sink connectors (e.g., DB to Kafka).
