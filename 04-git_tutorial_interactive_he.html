מבוא לבקרת גרסאות (VCS)

מערכת בקרת גרסאות היא אפליקציה שמנהלת ושומרת כל גרסה של הקוד והקבצים שלך.

🛡️

למה זה קריטי?

  • שיתוף פעולה על אותו קוד מקור בתוך צוות
  • ניהול גרסאות תוכנה
  • החזרה לאחור: חזרה לגרסאות קודמות אם מתגלים באגים
🌐

מודל מבוזר

בניגוד לכלים ישנים (כמו SVN), ב-Git כל מפתח יש לו עותק מלא של המאגר וכל ההיסטוריה שלו. זה מאפשר עבודה מהירה וללא חיבור לאינטרנט.

יתרונות ואתגרים של Git

✅

יתרונות

  • פיתוח מבוזר: עבודה על העותק המקומי שלך
  • ביצועים גבוהים: פעולות מהירות
  • קלות שימוש: פקודות אינטואיטיביות לאחר למידה
  • תמיכה חזקה בענפים: ניהול היסטוריה מעולה
  • כלים מתקדמים: פעולות מתקדמות בקלות
⚠️

אתגרים

  • מודל מחשבה חדש: גישה שונה לבקרת גרסאות
  • עקומת למידה: לוקח זמן להשתלט
  • פעולות הרסניות: חלק מהפקודות יכולות לגרום לאובדן מידע
  • מגבלות קבצים בינאריים: לא אידיאלי לקבצים בינאריים גדולים

מנגנון אחסון דלתות

Git משתמש במנגנון חכם כדי לחסוך מקום:

📦 קבצים מקוריים: מאוחסנים כאובייקט שלם בפעם הראשונה.

📝 שינויים: מאוחסנים כ-דלתות (הבדלים) בין גרסאות.

📸 Commit: "תמונת מצב" של הפרויקט ברגע נתון.

סוגי אובייקטים ב-Git

אובייקט בינארי גדול: מכיל את התוכן הגולמי של הקובץ (בינארי). אין לו מטא-דאטה (ללא שם קובץ, ללא הרשאות). רק נתונים טהורים.
מייצג תיקייה. מכיל רשימה של שמות קבצים ומצביעים ל-Blobs שלהם, או מצביעים ל-Trees אחרים (תת-תיקיות).
מייצג גרסה. מכיל מצביע ל-Tree הראשי של הפרויקט באותו רגע, בנוסף למטא-דאטה: יוצר, תאריך והודעה.
תווית המשויכת ל-commit, לדוגמה, מספר גרסה (v1.0, v2.0). Tags משמשים לסימון נקודות ספציפיות בהיסטוריה כחשובות.

פקודות בסיסיות

ערכת הכלים החיונית שכל מהנדס נתונים צריך לדעת.

git init # אתחול מאגר חדש בתיקייה הנוכחית
git clone <repo_url> # הורדת עותק מלא של מאגר מרוחק קיים
git status # בדיקת סטטוס קבצים (מה השתנה? מה מועבר?)
git add <file> # העברת קובץ לאזור ההעברה (הכנה לcommit)
git commit -m "message" # שמירת שינויים להיסטוריית המאגר המקומי

💡 טיפ מקצועי

השתמש ב-git diff לפני הוספת קבצים כדי לראות בדיוק אילו שורות השתנו, וב-git log כדי לצפות בהיסטוריה המלאה של הפרויקט.

תהליך העבודה (3 השלבים)

הבנת המבנה של Git היא המפתח למניעת שגיאות.

מאגר
היסטוריה שמורה
➜
אזור העברה
אינדקס (הכנה)
➜
תיקיית עבודה
סביבת עבודה (עריכה)

1. תיקיית עבודה

המקום בו אתה יוצר, עורך ומוחק קבצים. זהו סביבת העבודה המיידית שלך.

2. אזור העברה

שטח ביניים. אתה "בוחר" אילו קבצים שונו יכנסו לגרסה הבאה. git add מעביר קבצים לכאן.

3. מאגר

בסיס הנתונים. git commit לוקח הכל מאזור ההעברה ושומר אותו לנצח בהיסטוריה.

מצבי קבצים

  • לא במעקב: קובץ חדש ש-Git עדיין לא מכיר.
  • לא שונה: קובץ קיים שלא השתנה.
  • שונה: קובץ קיים עם שינויים.
  • מועבר: קובץ מסומן כמוכן ל-Commit הבא.
  • מוגש: שינויים נשמרו בבטחה במסד הנתונים המקומי.

הסתעפות ומיזוג

כוח העל של Git. עבודה על תכונות במקביל מבלי לשבור את הקוד הראשי.

# יצירת ענף חדש ומעבר אליו
git checkout -b feature_branch
# הצגת כל הענפים
git branch -a
# חזרה לענף הראשי
git checkout main
# מיזוג ענף צדדי לענף הראשי
git merge feature_branch

אסטרטגיות מיזוג

מיזוג Fast-Forward

קורה כאשר הענף הראשי לא השתנה מאז יצרת את הענף הצדדי. Git פשוט "מזיז את המצביע" קדימה. אין היסטוריה מתפצלת.

git merge --ff-only feature_branch # מיזוג רק אם fast-forward אפשרי

Merge Commit

קורה כאשר יש היסטוריה מתפצלת (גם הענף הראשי וגם הצדדי התקדמו). Git יוצר Merge Commit מיוחד שמחבר את שני הענפים.

git merge --no-ff feature_branch # תמיד צור merge commit

Squash Merge

משלב את כל ה-commits מהענף התכונה ל-commit בודד בענף היעד. מעולה להיסטוריה נקייה.

git merge --squash feature_branch
git commit -m "השלמת תכונה"

Octopus Merge

מיזוג יותר משני ענפים בו-זמנית. נדיר בשימוש, דורש שלא יהיו קונפליקטים.

git merge branch1 branch2 branch3

הדמיית מיזוג

הדמיה אינטראקטיבית להבנת אסטרטגיות מיזוג שונות.

ענף ראשי
ענף תכונה

ניהול קונפליקטים

מה קורה כאשר שני אנשים משנים את אותה שורה ב-אותו קובץ? Git עוצר ומבקש עזרה.

זיהוי קונפליקט

<<<<<<< HEAD
# קוד קיים בענף הראשי
print("Hello World")
=======
# קוד מהענף הצדדי
print("Hello Git")
>>>>>>> side_branch

איך לפתור?

  1. פתח את הקובץ
  2. בחר איזו גרסה לשמור (או שלב ידנית)
  3. מחק את הסימנים של Git (<<<<, ====, >>>>)
  4. הרץ git add על הקובץ המתוקן
  5. הרץ git commit או git merge --continue כדי לסיים את המיזוג

המלצה: השתמש ב-VSCode או IDE מודרני אחר. הוא מציג כפתורים מועילים כמו "קבל שינוי נכנס", "קבל שינוי נוכחי", "קבל את שני השינויים" שמקלים על התהליך.

שימוש בעורך המיזוג של VSCode

🔧 שלבי פתרון מיזוג ב-VSCode

  1. פתח את הקובץ עם הקונפליקט
  2. לחץ על "פתור בעורך המיזוג"
  3. השווה שינויים ופתור קונפליקטים
  4. בחר "קבל נכנס" לשינויים מהענף הצדדי
  5. בחר "קבל נוכחי" לשינויים מהענף הראשי
  6. עדכן את התוצאה ידנית במידת הצורך
  7. לחץ על "השלם מיזוג" כדי להעביר את הקובץ
  8. הרץ git merge --continue או השתמש ב-VSCode לסיום

רשימת פקודות מקיפה

ניהול מאגר

git init # יצירת מאגר מקומי חדש
git clone <url> # הורדת עותק של מאגר מרוחק
git status # הצגת סטטוס תיקיית העבודה
git remote -v # צפייה בכל המאגרים המרוחקים עם URLs
git remote # צפייה בכל המאגרים המרוחקים
git remote add origin <url> # הוספת מאגר מרוחק חדש
git remote add new_name <path> # הוספת שם חדש למאגר מרוחק

העברה ו-Commit

git add <file> # העברת קובץ ספציפי
git add . # העברת כל השינויים בתיקייה הנוכחית
git restore --staged <file> # ביטול העברה של קובץ
git restore --staged # ביטול העברה של כל הקבצים
git reset # ביטול העברת שינויים
git reset --hard # איפוס תיקיית העבודה ל-commit האחרון
git commit # יצירת commit חדש במאגר המקומי
git commit -m "message" # commit עם הודעה בשורה אחת

הסתעפות ומיזוג

git branch # הצגת כל הענפים
git branch <name> # יצירת ענף חדש
git branch -d <name> # מחיקת ענף מקומי (מונע אם לא מוזג)
git branch -D <name> # מחיקה כפויה של ענף (גם אם לא מוזג)
git branch -m old_name new_name # שינוי שם ענף
git checkout <branch> # מעבר לענף אחר
git checkout -b <name> # יצירה ומעבר לענף חדש (נכשל אם קיים)
git checkout -B <name> # יצירה או איפוס ענף קיים ל-HEAD הנוכחי
git merge <branch> # מיזוג ענף לענף הנוכחי
git merge <branch> --ff-only # מיזוג רק אם fast-forward אפשרי
git merge --no-ff <branch> # תמיד צור merge commit, גם אם fast-forward אפשרי
git merge --squash <branch> # שילוב כל ה-commits לאחד ללא merge commit
git merge -s octopus <branch1> <branch2> ... # מיזוג מספר ענפים עם merge commit בודד

Cherry-pick

git cherry-pick <commit-hash> # החלת commit מענף אחר
git cherry-pick <hash1> <hash2> # החלת מספר commits
git cherry-pick --continue # המשך לאחר פתרון קונפליקטים
git cherry-pick --abort # ביטול תהליך cherry-pick

היסטוריה ומידע

git log # הצגת היסטוריית commits במאגר המקומי
git log --oneline # היסטוריית commits קומפקטית
git log --oneline --graph --all # היסטוריה קומפקטית עם גרף ויזואלי של כל הענפים
git log --since="YYYY-MM-DD" --until="YYYY-MM-DD" # הצגת היסטוריית commits בטווח תאריכים
git show <commit> # הצגת פרטי commit ספציפי
git diff # השוואת שינויים (מקומי, מועבר, או בין commits)
git diff --staged # השוואת קבצים מועברים ל-commit האחרון
git blame <file> # הצגת מי שינה כל שורה בקובץ
git tag <tag-name> # יצירת תג מקומי ל-commit הנוכחי לסימון גרסה חשובה

ביטול שינויים

git checkout <file> # שחזור קובץ למצב ה-commit האחרון
git revert <commit> # יצירת commit חדש שמבטל שינויים מ-commit ספציפי
git revert <commit> --no-edit # ביטול ללא פתיחת עורך טקסט
git revert <commit> -n # ביטול ללא יצירת commit חדש
git reset HEAD <file> # הסרת קובץ ספציפי מאזור ההעברה
git reset --soft HEAD~1 # ביטול commit אחרון אך שמירת שינויים מועברים
git reset --mixed HEAD~1 # ביטול commit אחרון, שמירת שינויים בעבודה אך לא מועברים
git reset --hard HEAD~1 # ביטול commit אחרון ומחיקת שינויים

פקודות מתקדמות

כלים מתקדמים למשתמשי Git מנוסים.

Rebase

git rebase main # Rebase של הענף הנוכחי על main
git rebase -i master # Rebase אינטראקטיבי של הענף הנוכחי על master
# מאפשר לערוך, לשלב או להסיר commits
# פקודות rebase אינטראקטיבי:
# p, pick = שימוש ב-commit כמו שהוא
# r, reword = שימוש ב-commit, אך עריכת הודעת ה-commit
# e, edit = שימוש ב-commit, אך עצירה לתיקון
# s, squash = שילוב commit לקודם (שמירת שתי ההודעות)
# f, fixup = שילוב commit לקודם (השלכת הודעה זו)
# x, exec = הרצת פקודת shell במהלך rebase
# d, drop = הסרת commit לחלוטין

Bisect

git bisect # התחלת חיפוש בינארי למציאת באג
git bisect start # התחלת סשן bisect
git bisect bad # סימון commit נוכחי כרע
git bisect good <commit-hash> # סימון commit ידוע כטוב
git bisect run <script_name> # הרצת בדיקה אוטומטית עם סקריפט
git bisect reset # סיום bisect וחזרה למצב המקורי

Filter Repo

git install git-filter-repo # התקנת כלי filter-repo
git filter-repo --path secret.txt --invert-paths # הסרת קובץ ספציפי מההיסטוריה
git filter-repo --path config.ini --invert-paths --force # הסרה כפויה של קובץ מההיסטוריה

⚠️ אזהרה: זה משכתב את היסטוריית Git. השתמש בזהירות רבה!

Reflog (שחזור)

git reflog # הצגת יומן הפניות
git reflog show # הצגת פרטי reflog
git reflog expire # ניקוי רשומות reflog ישנות
git reflog --since # הצגת רשומות reflog מזמן ספציפי
git reflog --until # הצגת רשומות reflog עד זמן ספציפי

Stash

git stash # שמירת שינויים זמנית
git stash list # הצגת כל ה-stashes
git stash pop # החלה והסרה של stash אחרון
git stash apply # החלת stash אחרון (שמירתו)
git stash drop # מחיקת stash אחרון

Worktree

git worktree # ניהול מספר תיקיות עבודה
git worktree add <path> <branch> # יצירת worktree חדש בנתיב לענף
git worktree list # הצגת כל ה-worktrees המקושרים
git worktree remove <path> # הסרת worktree

Git Submodules

git submodule # ניהול submodules
git submodule add <repository_link|dir> <path> # הוספת submodule חדש
git submodule list # הצגת submodules
git submodule status # הצגת סטטוס (commit + מצב סינכרון) של submodules
git submodule update --init # אתחול ומשיכת submodules
git submodule update --init --recursive # אתחול ועדכון כל ה-submodules המקוננים
git submodule update --init <path_to_submodule> # אתחול ועדכון submodule ספציפי
git submodule deinit <submodule_name> # ביטול רישום submodule
git rm <path> # הסרת submodule מהמאגר

תרגילים מעשיים

הדרך הטובה ביותר ללמוד היא על ידי עשייה. פתח את הטרמינל שלך.

📝 תרגיל 1: מאגר מקומי

  1. צור תיקייה חדשה ופתח טרמינל
  2. אתחל מאגר: git init
  3. צור קובץ name.txt עם השם שלך
  4. העבר אותו: git add name.txt
  5. בצע commit: git commit -m "Initial commit"

🌿 תרגיל 2: הסתעפות

  1. צור ענף חדש: git checkout -b side_branch
  2. שנה את תוכן name.txt לשם של חבר
  3. העבר ובצע commit של השינויים בענף החדש
  4. בדוק היסטוריה: git log ו-git status

🔀 תרגיל 3: מיזוג וקונפליקטים

  1. חזור לענף הראשי: git checkout main
  2. שנה את name.txt גם בענף הראשי (צור קונפליקט)
  3. העבר ובצע commit של השינוי
  4. נסה למזג: git merge side_branch
  5. שים לב: יש לך קונפליקט!
  6. פתור את הקונפליקט כך ששני השמות יופיעו
  7. העבר את הקובץ הפתור וסיים את המיזוג עם git commit

🍒 תרגיל 4: Cherry-pick

  1. צור ענף חדש: git checkout -b experimental
  2. בצע מספר commits עם שינויים שונים
  3. חזור לענף הראשי: git checkout main
  4. השתמש ב-git log --oneline experimental כדי למצוא hash של commit
  5. בצע cherry-pick של commit ספציפי: git cherry-pick <hash>
