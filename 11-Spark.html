HDFS - Hadoop Distributed File System

עקרונות עבודה

  • קבצים מפוצלים ל-Blocks (ברירת מחדל: 128MB)
  • כל Block משוכפל על מספר DataNodes (ברירת מחדל: 3 עותקים)
  • NameNode מאחסן את המטא-דאטה
  • DataNodes מאחסנים את הנתונים בפועל
# דוגמה: מבנה HDFS NameNode (Metadata): /logs/031512.log: B1,B2,B3 /logs/042313.log: B4,B5 Block Locations: B1: NodeA, NodeB, NodeD B2: NodeB, NodeD, NodeE B3: NodeA, NodeB, NodeC B4: NodeA, NodeB, NodeE B5: NodeC, NodeE, NodeD

YARN - Yet Another Resource Negotiator

תפקידים

  • Resource Manager - מנהל משאבים ראשי
  • Node Manager - מנהל משאבים בכל Node
  • Application Master - מנהל יישום ספציפי
  • Container - יחידת הרצה עם משאבים מוקצים

⚙️ יתרונות YARN

  • אפשרות להריץ מספר Jobs במקביל
  • אינטגרציה עם HDFS לביצועים טובים
  • ניהול משאבים יעיל

ZooKeeper

שימושים

  • Centralized Configuration - ניהול תצורה מרכזי
  • Naming Service - שירות שמות
  • Synchronization - סנכרון בין processes
  • Voting System - מערכת הצבעה

🔥 RDD - Resilient Distributed Dataset

מבנה הנתונים הבסיסי של Spark

● מהו RDD?

Resilient Distributed Dataset

  • Resilient - עמיד לכשלים, יכול להתאושש
  • Distributed - מבוזר על פני Cluster
  • Dataset - אוסף נתונים

חשוב: RDD הוא Immutable (לא ניתן לשינוי)!

● איך יוצרים RDD?

3 דרכים ליצירת RDD:

1️⃣ מקובץ או תיקייה

# קריאה מ-path יחסי (משתמש ב-default fs configuration) rdd = sc.textFile('/data/sample/table1/') # קריאה מ-HDFS rdd = sc.textFile('hdfs://my-hdfs:8020/data/sample/table1/') # קריאה מ-local filesystem rdd = sc.textFile('file:///home/my_user/my_data/') # קריאה מ-S3 rdd = sc.textFile('s3a://spark/data/sample/table1/') # קריאה רק של קבצי CSV rdd = sc.textFile('/data/sample/table1/*.csv')

2️⃣ מנתונים בזיכרון

# Python - Regular RDD data = [7, 17, 25, 88, 28] rdd = sc.parallelize(data) # Python - Key-Value RDD data = [('avi', 40), ('yossi', 17), ('eli', 8)] rdd = sc.parallelize(data)
// Java - Regular RDD List data = Arrays.asList(7, 17, 25, 88, 28); JavaRDD rdd = sc.parallelize(data); // Java - Key-Value RDD List> data = Arrays.asList( new Tuple2<>("avi", 40), new Tuple2<>("yossi", 17) ); JavaPairRDD rdd = sc.parallelizePairs(data);

3️⃣ מ-RDD אחר

באמצעות Transformations על RDD קיים, נוצר RDD חדש.

● Transformations vs Actions

שני סוגי פעולות ב-RDD

Transformation - תמיד רץ על ה-Cluster

Action - יכול להחזיר ערך ל-Driver או לכתוב ל-Data Store

🔄 Transformation

יוצר RDD חדש

Lazy - לא מתבצע מיד

דוגמאות: map, filter, flatMap

⚡ Action

מחזיר ערך או כותב

מפעיל ביצוע בפועל

דוגמאות: count, collect, save

Transformations נפוצות:

# map - מיפוי כל אלמנט rdd.map(lambda x: x * 2) # filter - סינון rdd.filter(lambda x: x > 10) # flatMap - מיפוי והשטחה rdd.flatMap(lambda line: line.split()) # reduceByKey - צמצום לפי מפתח (Pair RDD) rdd.reduceByKey(lambda a, b: a + b) # sortByKey - מיון לפי מפתח rdd.sortByKey() # distinct - ערכים ייחודיים rdd.distinct() # union - איחוד שני RDDs rdd1.union(rdd2) # join - חיבור שני Pair RDDs rdd1.join(rdd2)

Actions נפוצות:

# count - ספירת אלמנטים count = rdd.count() # collect - החזרת כל האלמנטים ל-Driver data = rdd.collect() # take - החזרת N אלמנטים ראשונים first_5 = rdd.take(5) # first - החזרת אלמנט ראשון first = rdd.first() # reduce - צמצום לערך יחיד sum = rdd.reduce(lambda a, b: a + b) # foreach - ביצוע פעולה על כל אלמנט rdd.foreach(lambda x: print(x)) # saveAsTextFile - שמירה לקובץ rdd.saveAsTextFile('/output/path')

● Partitioning

מהו Partition?

RDD מחולק ל-Partitions - חלקים לוגיים של הנתונים.

כל Partition מעובד על Executor שונה.

למה חשוב?

  • ביצועים - מספר Partitions משפיע על מקבילות
  • זיכרון - כל Partition צריך להיכנס לזיכרון Executor
  • Shuffle - פחות Shuffle = ביצועים טובים יותר
# בדיקת מספר Partitions num_partitions = rdd.getNumPartitions() # שינוי מספר Partitions - coalesce (רק הקטנה) rdd = rdd.coalesce(2) # שינוי מספר Partitions - repartition (הגדלה/הקטנה) rdd = rdd.repartition(10) # יצירת RDD עם מספר Partitions מוגדר rdd = sc.textFile('/path', minPartitions=10)

● Broadcast & Accumulators

Broadcast Variables

Broadcast מאפשר לשתף משתנה read-only עם כל ה-Executors ביעילות.

במקום לשלוח את המשתנה עם כל Task, הוא נשלח פעם אחת לכל Executor.

# יצירת Broadcast Variable lookup_table = {"key1": "value1", "key2": "value2"} broadcast_lookup = sc.broadcast(lookup_table) # שימוש ב-Broadcast Variable def enrich_data(x): return broadcast_lookup.value.get(x, "unknown") enriched = rdd.map(enrich_data)

Accumulators

Accumulator הוא משתנה שניתן רק להוסיף לו ערכים (ב-Executors) ולקרוא (ב-Driver).

שימושי לספירות ו-sums על פני כל ה-Cluster.

# יצירת Accumulator error_counter = sc.accumulator(0) def process_line(line): global error_counter try: # process line return processed_line except: error_counter.add(1) return None processed = rdd.map(process_line) processed.count() # מפעיל את החישוב print(f"Total errors: {error_counter.value}")

● Spark Streaming (DStreams)

⚠️ הערה

DStreams הוא ה-API הישן של Spark Streaming.

# Python - Spark Streaming עם DStreams from pyspark import SparkContext from pyspark.streaming import StreamingContext sc = SparkContext("local[2]", "NetworkWordCount") ssc = StreamingContext(sc, 1) # batch interval של 1 שניה # קריאה מ-socket lines = ssc.socketTextStream("localhost", 9999) # עיבוד words = lines.flatMap(lambda line: line.split()) pairs = words.map(lambda word: (word, 1)) wordCounts = pairs.reduceByKey(lambda a, b: a + b) wordCounts.pprint() # התחלה ssc.start() ssc.awaitTermination()

● Structured Streaming

✅ ה-API המומלץ!

Structured Streaming מבוסס על DataFrame/Dataset API.

גישה אחידה לעיבוד batch ו-streaming!

קריאה מ-Kafka

# Python - Structured Streaming עם Kafka from pyspark.sql import SparkSession spark = SparkSession.builder \ .appName("StructuredKafkaWordCount") \ .getOrCreate() # קריאה מ-Kafka df = spark \ .readStream \ .format("kafka") \ .option("kafka.bootstrap.servers", "localhost:9092") \ .option("subscribe", "my-topic") \ .load() # הנתונים מגיעים כ-binary, צריך להמיר from pyspark.sql.functions import col lines = df.select(col("value").cast("string")) # עיבוד words = lines.selectExpr("explode(split(value, ' ')) as word") wordCounts = words.groupBy("word").count() # כתיבה ל-console query = wordCounts \ .writeStream \ .outputMode("complete") \ .format("console") \ .start() query.awaitTermination()

Caching & Persistence

# Cache ב-memory df.cache() # או df.persist() # Cache עם storage level from pyspark import StorageLevel df.persist(StorageLevel.MEMORY_AND_DISK) # ניקוי cache df.unpersist()

Storage Levels:

  • MEMORY_ONLY - רק זיכרון (default)
  • MEMORY_AND_DISK - זיכרון + דיסק
  • DISK_ONLY - רק דיסק
  • MEMORY_ONLY_SER - זיכרון מסיריאליזד

Rolling Logs

● Graceful Shutdown

⚠️ הבעיה עם SIGTERM

כאשר שולחים SIGTERM ל-Spark:

  • כל התהליכים נעצרים מיד
  • Threads לא נסגרים כראוי
  • נתונים עלולים ללכת לאיבוד

Graceful Shutdown - Configuration

# הוספת configuration spark.streaming.stopGracefullyOnShutdown = true

Shutdown Listener

# Python - Graceful Shutdown import atexit from time import sleep def graceful_shutdown(spark, query_stream): """עצירה חלקה של Streaming""" query_stream.stop() while query_stream.isActive(): sleep(5) spark.stop() # רישום ה-shutdown handler atexit.register(graceful_shutdown, spark, query_stream)

עם While Loop במקום awaitTermination

# במקום awaitTermination # query.awaitTermination() # השתמש ב-while loop import signal import sys stop_flag = False def signal_handler(sig, frame): global stop_flag print("Stopping gracefully...") stop_flag = True signal.signal(signal.SIGTERM, signal_handler) signal.signal(signal.SIGINT, signal_handler) while not stop_flag: if not query.isActive: break sleep(1) graceful_shutdown(spark, query)

✅ יתרונות Graceful Shutdown

  • סיום נקי של כל ה-batches
  • שמירת checkpoints
  • סגירה נכונה של connections
  • אין אובדן נתונים
