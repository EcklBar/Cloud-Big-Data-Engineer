🔥 RDD - Resilient Distributed Dataset

מבנה הנתונים הבסיסי של Spark

● מהו RDD?

Resilient Distributed Dataset

  • Resilient - עמיד לכשלים, יכול להתאושש
  • Distributed - מבוזר על פני Cluster
  • Dataset - אוסף נתונים

חשוב: RDD הוא Immutable (לא ניתן לשינוי)!

● איך יוצרים RDD?

3 דרכים ליצירת RDD:

1️⃣ מקובץ או תיקייה

# קריאה מ-path יחסי (משתמש ב-default fs configuration) rdd = sc.textFile('/data/sample/table1/') # קריאה מ-HDFS rdd = sc.textFile('hdfs://my-hdfs:8020/data/sample/table1/') # קריאה מ-local filesystem rdd = sc.textFile('file:///home/my_user/my_data/') # קריאה מ-S3 rdd = sc.textFile('s3a://spark/data/sample/table1/') # קריאה רק של קבצי CSV rdd = sc.textFile('/data/sample/table1/*.csv')

2️⃣ מנתונים בזיכרון

# Python - Regular RDD data = [7, 17, 25, 88, 28] rdd = sc.parallelize(data) # Python - Key-Value RDD data = [('avi', 40), ('yossi', 17), ('eli', 8)] rdd = sc.parallelize(data)
// Java - Regular RDD List data = Arrays.asList(7, 17, 25, 88, 28); JavaRDD rdd = sc.parallelize(data); // Java - Key-Value RDD List> data = Arrays.asList( new Tuple2<>("avi", 40), new Tuple2<>("yossi", 17) ); JavaPairRDD rdd = sc.parallelizePairs(data);

● Lazy Evaluation

מהי Lazy Evaluation?

Spark לא מבצע Transformations מיד - הוא בונה תוכנית ביצוע (DAG) ומבצע אותה רק כאשר Action נקרא.

✅ יתרונות

  • אופטימיזציה - Spark יכול לאפטם את כל התוכנית לפני הביצוע
  • יעילות - הימנעות מחישובים מיותרים
  • Pipeline Optimization - איחוד פעולות
# כל השורות האלו לא מבצעות כלום! rdd1 = sc.textFile('/data/input.txt') # Lazy rdd2 = rdd1.filter(lambda x: len(x) > 0) # Lazy rdd3 = rdd2.map(lambda x: x.upper()) # Lazy # רק כאן Spark מתחיל לעבוד בפועל count = rdd3.count() # Action - מפעיל את כל הביצוע!

⚠️ חשוב לזכור

אם תקרא ל-Action פעמיים על אותו RDD, החישוב יתבצע פעמיים!

לכן כדאי להשתמש ב-cache() אם אתה צריך את התוצאה יותר מפעם אחת.

● Pair RDD (Key-Value)

מהו Pair RDD?

Pair RDD הוא RDD של זוגות (Key, Value).

מאפשר פעולות מתקדמות כמו groupByKey, reduceByKey, join ועוד.

# יצירת Pair RDD data = [('apple', 5), ('banana', 3), ('apple', 2), ('orange', 8)] pair_rdd = sc.parallelize(data) # או המרה מ-RDD רגיל lines = sc.textFile('/data/users.txt') # כל שורה: "John,25" pairs = lines.map(lambda line: (line.split(',')[0], int(line.split(',')[1]))) # תוצאה: [('John', 25), ('Mary', 30), ...]

פעולות על Pair RDD

# reduceByKey - צמצום לפי מפתח (שומר על Partitioning!) word_counts = pairs.reduceByKey(lambda a, b: a + b) # groupByKey - קיבוץ ערכים לפי מפתח grouped = pairs.groupByKey() # mapValues - map רק על הערכים doubled = pairs.mapValues(lambda x: x * 2) # keys / values - קבלת רק המפתחות או הערכים all_keys = pairs.keys() all_values = pairs.values() # sortByKey - מיון לפי מפתח sorted_pairs = pairs.sortByKey(ascending=True) # join - חיבור שני Pair RDDs rdd1.join(rdd2) # leftOuterJoin / rightOuterJoin rdd1.leftOuterJoin(rdd2)

⚠️ groupByKey vs reduceByKey

groupByKey - מעביר את כל הנתונים דרך הרשת (Shuffle כבד!)

reduceByKey - מבצע צמצום מקומי לפני Shuffle (יעיל יותר!)

# ❌ לא יעיל - groupByKey pairs.groupByKey().mapValues(lambda values: sum(values)) # ✅ יעיל - reduceByKey pairs.reduceByKey(lambda a, b: a + b)

● Transformations vs Actions

שני סוגי פעולות ב-RDD

Transformation - תמיד רץ על ה-Cluster, Lazy Evaluation

Action - מפעיל ביצוע בפועל, יכול להחזיר ערך ל-Driver או לכתוב ל-Data Store

🔄 Transformation

יוצר RDD חדש

Lazy - לא מתבצע מיד

דוגמאות: map, filter, flatMap

⚡ Action

מחזיר ערך או כותב

מפעיל ביצוע בפועל

דוגמאות: count, collect, save

Transformations נפוצות:

# map - מיפוי כל אלמנט (1:1) rdd = sc.parallelize([1, 2, 3, 4]) rdd.map(lambda x: x * 2) # [2, 4, 6, 8] # filter - סינון אלמנטים rdd.filter(lambda x: x > 10) # רק אלמנטים גדולים מ-10 # flatMap - מיפוי והשטחה (1:N) lines = sc.parallelize(["hello world", "spark rocks"]) words = lines.flatMap(lambda line: line.split()) # ["hello", "world", "spark", "rocks"] # mapPartitions - עיבוד partition שלם בבת אחת (יעיל!) def process_partition(iterator): # פעולה יקרה שנעשית פעם אחת per partition connection = create_db_connection() for record in iterator: yield process_record(connection, record) connection.close() rdd.mapPartitions(process_partition) # mapPartitionsWithIndex - כמו mapPartitions עם מספר partition def process_with_index(partition_index, iterator): yield f"Partition {partition_index}: {sum(iterator)}" rdd.mapPartitionsWithIndex(process_with_index) # sample - דגימה אקראית rdd.sample(withReplacement=False, fraction=0.1) # 10% דגימה # distinct - ערכים ייחודיים בלבד rdd.distinct() # union - איחוד שני RDDs (כולל כפילויות) rdd1.union(rdd2) # intersection - חיתוך (ערכים משותפים) rdd1.intersection(rdd2) # subtract - הפרש (ערכים ב-rdd1 שלא ב-rdd2) rdd1.subtract(rdd2) # === Pair RDD Transformations === # reduceByKey - צמצום לפי מפתח (יעיל!) pairs.reduceByKey(lambda a, b: a + b) # groupByKey - קיבוץ ערכים לפי מפתח (פחות יעיל) pairs.groupByKey() # sortByKey - מיון לפי מפתח pairs.sortByKey(ascending=True) # mapValues - map רק על הערכים pairs.mapValues(lambda x: x * 2) # join - חיבור שני Pair RDDs rdd1.join(rdd2) # Inner Join rdd1.leftOuterJoin(rdd2) rdd1.rightOuterJoin(rdd2) # cogroup - קיבוץ משני RDDs rdd1.cogroup(rdd2)

Actions נפוצות:

# count - ספירת אלמנטים count = rdd.count() # מחזיר מספר # collect - החזרת כל האלמנטים ל-Driver (זהירות! רק לנתונים קטנים) data = rdd.collect() # מחזיר list # take - החזרת N אלמנטים ראשונים first_5 = rdd.take(5) # מחזיר list של 5 אלמנטים # takeSample - דגימה אקראית sample = rdd.takeSample(withReplacement=False, num=10) # takeOrdered - N אלמנטים הקטנים ביותר (ממויינים) smallest_5 = rdd.takeOrdered(5) largest_5 = rdd.takeOrdered(5, key=lambda x: -x) # הפוך # first - החזרת אלמנט ראשון first = rdd.first() # reduce - צמצום לערך יחיד sum_all = rdd.reduce(lambda a, b: a + b) max_value = rdd.reduce(lambda a, b: a if a > b else b) # fold - כמו reduce אבל עם ערך התחלתי total = rdd.fold(0, lambda acc, value: acc + value) # aggregate - aggregation מתקדם # aggregate(zeroValue, seqOp, combOp) sum_and_count = rdd.aggregate( (0, 0), # (sum, count) lambda acc, value: (acc[0] + value, acc[1] + 1), # per partition lambda acc1, acc2: (acc1[0] + acc2[0], acc1[1] + acc2[1]) # combine ) average = sum_and_count[0] / sum_and_count[1] # foreach - ביצוע פעולה על כל אלמנט (ב-Executors) rdd.foreach(lambda x: print(x)) # יודפס ב-Executors, לא ב-Driver! # foreachPartition - ביצוע פעולה על partition שלם def save_partition(iterator): connection = create_db_connection() for record in iterator: connection.insert(record) connection.close() rdd.foreachPartition(save_partition) # === פעולות שמירה === # saveAsTextFile - שמירה כקבצי טקסט rdd.saveAsTextFile('/output/path') # saveAsSequenceFile - Hadoop SequenceFile (Pair RDD) pair_rdd.saveAsSequenceFile('/output/path') # === Pair RDD Actions === # countByKey - ספירת ערכים לכל מפתח counts = pair_rdd.countByKey() # dict: {key: count} # collectAsMap - המרה ל-dictionary (זהירות! רק לנתונים קטנים) result_dict = pair_rdd.collectAsMap() # lookup - קבלת כל הערכים למפתח מסוים values = pair_rdd.lookup('key1') # list של ערכים

● Partitioning

מהו Partition?

RDD מחולק ל-Partitions - חלקים לוגיים של הנתונים.

כל Partition מעובד על Executor שונה.

למה חשוב?

  • ביצועים - מספר Partitions משפיע על מקבילות
  • זיכרון - כל Partition צריך להיכנס לזיכרון Executor
  • Shuffle - פחות Shuffle = ביצועים טובים יותר
# בדיקת מספר Partitions num_partitions = rdd.getNumPartitions() # שינוי מספר Partitions - coalesce (רק הקטנה) rdd = rdd.coalesce(2) # שינוי מספר Partitions - repartition (הגדלה/הקטנה) rdd = rdd.repartition(10) # יצירת RDD עם מספר Partitions מוגדר rdd = sc.textFile('/path', minPartitions=10)

● Broadcast & Accumulators

Broadcast Variables

Broadcast מאפשר לשתף משתנה read-only עם כל ה-Executors ביעילות.

במקום לשלוח את המשתנה עם כל Task, הוא נשלח פעם אחת לכל Executor.

# יצירת Broadcast Variable lookup_table = {"key1": "value1", "key2": "value2"} broadcast_lookup = sc.broadcast(lookup_table) # שימוש ב-Broadcast Variable def enrich_data(x): return broadcast_lookup.value.get(x, "unknown") enriched = rdd.map(enrich_data)

Accumulators

Accumulator הוא משתנה שניתן רק להוסיף לו ערכים (ב-Executors) ולקרוא (ב-Driver).

שימושי לספירות ו-sums על פני כל ה-Cluster.

# יצירת Accumulator error_counter = sc.accumulator(0) def process_line(line): global error_counter try: # process line return processed_line except: error_counter.add(1) return None processed = rdd.map(process_line) processed.count() # מפעיל את החישוב print(f"Total errors: {error_counter.value}")

● 📋 Quick Reference - RDD Commands

🔄 Transformations (Lazy)

פעולה תיאור דוגמה
map(func) מיפוי 1:1 - מחזיר RDD חדש עם func על כל אלמנט rdd.map(lambda x: x * 2)
filter(func) סינון - מחזיר רק אלמנטים שעוברים את התנאי rdd.filter(lambda x: x > 10)
flatMap(func) מיפוי 1:N + השטחה - כל אלמנט יכול להחזיר 0 או יותר rdd.flatMap(lambda x: x.split())
mapPartitions(func) עיבוד partition שלם (יעיל!) rdd.mapPartitions(process_part)
distinct() ערכים ייחודיים בלבד rdd.distinct()
union(other) איחוד שני RDDs (כולל כפילויות) rdd1.union(rdd2)
intersection(other) חיתוך - ערכים משותפים rdd1.intersection(rdd2)

מהי UDF?

UDF מאפשרת להגדיר פונקציה מותאמת אישית ולהשתמש בה ב-DataFrames ו-SQL.

# Python UDF - דרך 1 from pyspark.sql.functions import udf from pyspark.sql.types import StringType def categorize_age(age): if age < 18: return "Minor" elif age < 65: return "Adult" else: return "Senior" # רישום UDF categorize_udf = udf(categorize_age, StringType()) # שימוש df.withColumn("age_category", categorize_udf(col("age"))) # Python UDF - דרך 2 (עם decorator) @udf(returnType=StringType()) def categorize_age_udf(age): if age < 18: return "Minor" elif age < 65: return "Adult" else: return "Senior" df.withColumn("age_category", categorize_age_udf(col("age")))
# רישום UDF ל-SQL spark.udf.register("categorize_age", categorize_age, StringType()) # שימוש ב-SQL spark.sql(""" SELECT name, age, categorize_age(age) as age_category FROM users """)

● Window Functions

מהן Window Functions?

Window Functions מאפשרות לבצע חישובים על "חלון" של שורות.

דומה ל-SQL Window Functions.

from pyspark.sql.window import Window from pyspark.sql.functions import row_number, rank, dense_rank, lag, lead # הגדרת Window window_spec = Window.partitionBy("department").orderBy("salary") # שימוש ב-Window Functions df.withColumn("row_num", row_number().over(window_spec)) \ .withColumn("rank", rank().over(window_spec)) \ .withColumn("dense_rank", dense_rank().over(window_spec)) # Lag / Lead - גישה לשורות קודמות/הבאות df.withColumn("prev_salary", lag("salary", 1).over(window_spec)) \ .withColumn("next_salary", lead("salary", 1).over(window_spec)) # Ntile - חלוקה לקבוצות (quartiles, percentiles) from pyspark.sql.functions import ntile df.withColumn("quartile", ntile(4).over(window_spec)) \ .withColumn("decile", ntile(10).over(window_spec)) # Percent Rank - אחוזון יחסי (0 עד 1) from pyspark.sql.functions import percent_rank df.withColumn("percent_rank", percent_rank().over(window_spec)) # Aggregations על Window from pyspark.sql.functions import sum, avg, max, min window_spec2 = Window.partitionBy("department") \ .orderBy("date") \ .rowsBetween(Window.unboundedPreceding, Window.currentRow) df.withColumn("running_total", sum("amount").over(window_spec2)) \ .withColumn("running_avg", avg("amount").over(window_spec2)) # Top N per group window_spec3 = Window.partitionBy("department").orderBy(col("salary").desc()) top_earners = df.withColumn("rank", rank().over(window_spec3)) \ .filter(col("rank") <= 3)

סוגי Window Frames

  • rowsBetween - מבוסס על מספר שורות
  • rangeBetween - מבוסס על ערכים
  • unboundedPreceding - מתחילת ה-partition
  • unboundedFollowing - עד סוף ה-partition
  • currentRow - השורה הנוכחית

● Catalyst Optimizer

מהו Catalyst?

ארכיטקטורת Kafka

  • Broker - שרת Kafka (בדרך כלל כמה brokers ב-cluster)

  • Consumer Group - קבוצת Consumers שעובדים ביחד
  • ZooKeeper - ניהול metadata (עד Kafka 2.8, אחר כך KRaft)

יתרונות Kafka

  • Throughput גבוה - מיליוני הודעות לשנייה
  • Low Latency - מילישניות
  • Durability - הודעות נשמרות לדיסק
  • Scalability - הוספת brokers/partitions
  • Fault Tolerance - שכפול (replication)
  • Ordering - שמירת סדר per partition

איך זה עובד?

Stateful Streaming - ניהול State

מהו State?

State מאפשר לזכור מידע בין batches - לדוגמה, aggregations running.

# דוגמה - running count per word wordCounts = words \ .groupBy("word") \ .count() # State: שומר count עבור כל word # State נשמר ב-checkpoint! query = wordCounts \ .writeStream \ .outputMode("update") \ .option("checkpointLocation", "/checkpoint") \ .format("console") \ .start()
# mapGroupsWithState - custom state management from pyspark.sql.streaming import GroupState def update_state(key, values, state): # key = המפתח שאנחנו מקבצים לפיו # values = ערכים חדשים ב-batch הנוכחי # state = ה-state הקודם if state.exists: old_count = state.get else: old_count = 0 new_count = old_count + sum(values) state.update(new_count) return (key, new_count) # שימוש result = df.groupByKey(lambda x: x.key) \ .mapGroupsWithState(update_state)

● Graceful Shutdown

⚠️ הבעיה עם SIGTERM

כאשר שולחים SIGTERM ל-Spark:

  • כל התהליכים נעצרים מיד
  • Threads לא נסגרים כראוי
  • נתונים עלולים ללכת לאיבוד

Graceful Shutdown - Configuration

# הוספת configuration spark.streaming.stopGracefullyOnShutdown = true

Shutdown Listener

# Python - Graceful Shutdown import atexit from time import sleep def graceful_shutdown(spark, query_stream): """עצירה חלקה של Streaming""" query_stream.stop() while query_stream.isActive(): sleep(5) spark.stop() # רישום ה-shutdown handler atexit.register(graceful_shutdown, spark, query_stream)

עם While Loop במקום awaitTermination

# במקום awaitTermination # query.awaitTermination() # השתמש ב-while loop import signal import sys stop_flag = False def signal_handler(sig, frame): global stop_flag print("Stopping gracefully...") stop_flag = True signal.signal(signal.SIGTERM, signal_handler) signal.signal(signal.SIGINT, signal_handler) while not stop_flag: if not query.isActive: break sleep(1) graceful_shutdown(spark, query)

✅ יתרונות Graceful Shutdown

  • סיום נקי של כל ה-batches
  • שמירת checkpoints
  • סגירה נכונה של connections
  • אין אובדן נתונים

⭐ Best Practices

עקרונות ושיטות עבודה מומלצות

💾 פורמטים וקבצים

✅ DO - השתמש ב-Parquet

  • Parquet = ברירת מחדל! - מהיר פי 10-100 מ-CSV
  • Columnar format - קריאה רק של עמודות נדרשות
  • דחיסה מעולה - חוסך מקום ורוחב פס
  • Schema מובנה - לא צריך inferSchema
# ✅ Write Parquet with Partitioning df.write \ .partitionBy("year", "month", "day") \ .mode("overwrite") \ .parquet("/data/output") # ✅ Read Parquet - רק נתונים רלוונטיים! df = spark.read.parquet("/data/output") \ .filter("year=2023 AND month=12")

❌ DON'T - אל תשתמש ב-CSV בפרודקשן

  • איטי מאוד
  • גדול (לא דחוס)
  • צריך inferSchema או schema ידני
  • בעיות עם encoding, delimiters, quotes

🎯 Golden Rules

  1. Push Down - filter/select מוקדם ככל האפשר
  2. Avoid Shuffle - broadcast, partitionBy, coalesce
  3. Cache Smart - רק נתונים שנמשכים 2+ פעמים
  4. Right Partitions - 2-4 partitions לכל core
  5. Parquet Always - לא CSV!

תרגיל 1
RDD - ספירת מילים (Word Count)

משימה: קרא קובץ טקסט וספור כמה פעמים כל מילה מופיעה.

  • קרא קובץ טקסט באמצעות sc.textFile()
  • פצל כל שורה למילים
  • המר כל מילה ל-(word, 1)
  • צמצם לפי מפתח לספירת המילים
  • הצג את 10 המילים הנפוצות ביותר
# פתרון lines = sc.textFile("/path/to/file.txt") words = lines.flatMap(lambda line: line.split()) word_pairs = words.map(lambda word: (word.lower(), 1)) word_counts = word_pairs.reduceByKey(lambda a, b: a + b) top_10 = word_counts.sortBy(lambda x: x[1], ascending=False).take(10) for word, count in top_10: print(f"{word}: {count}")
תרגיל 2
DataFrame - ניתוח נתוני מכירות

משימה: נתח נתוני מכירות מקובץ CSV.

  • קרא קובץ CSV עם עמודות: product, category, price, quantity, date
  • סנן רק מכירות מ-2023
  • הוסף עמודה: total_price = price * quantity
  • מצא את סך המכירות לפי קטגוריה
  • מיין לפי סך מכירות (יורד)
# פתרון from pyspark.sql.functions import col, year, sum as _sum # קריאת הנתונים df = spark.read.csv("/path/to/sales.csv", header=True, inferSchema=True) # סינון ל-2023 df_2023 = df.filter(year(col("date")) == 2023) # הוספת total_price df_with_total = df_2023.withColumn("total_price", col("price") * col("quantity")) # קיבוץ לפי קטגוריה sales_by_category = df_with_total.groupBy("category") \ .agg(_sum("total_price").alias("total_sales")) # מיון result = sales_by_category.orderBy(col("total_sales").desc()) result.show()
תרגיל 3
Joins - חיבור טבלאות

משימה: חבר בין טבלת משתמשים לטבלת הזמנות.

  • טבלה 1: users (id, name, city)
  • טבלה 2: orders (order_id, user_id, amount)
  • חבר את הטבלאות
  • חשב סך הזמנות לכל משתמש
  • הצג משתמשים ללא הזמנות (אם יש)
# פתרון from pyspark.sql.functions import sum, coalesce users = spark.createDataFrame([ (1, "Alice", "Tel Aviv"), (2, "Bob", "Haifa"), (3, "Charlie", "Jerusalem") ], ["id", "name", "city"]) orders = spark.createDataFrame([ (101, 1, 100), (102, 1, 200), (103, 2, 150) ], ["order_id", "user_id", "amount"]) # Left Join - כולל משתמשים ללא הזמנות result = users.join(orders, users.id == orders.user_id, "left") # קיבוץ וסיכום from pyspark.sql.functions import lit user_totals = result.groupBy("id", "name", "city") \ .agg( coalesce(sum("amount"), lit(0)).alias("total_orders") ) user_totals.show() # משתמשים ללא הזמנות no_orders = user_totals.filter(col("total_orders") == 0) no_orders.show()
תרגיל 4
Window Functions - דירוג עובדים

משימה: דרג עובדים לפי משכורת בכל מחלקה.

  • טבלה: employees (name, department, salary)
  • הוסף עמודת rank לפי משכורת בכל מחלקה
  • מצא את 3 המשכורות הגבוהות בכל מחלקה
  • חשב משכורת ממוצעת per מחלקה והוסף כעמודה
# פתרון from pyspark.sql.window import Window from pyspark.sql.functions import rank, avg employees = spark.createDataFrame([ ("Alice", "Engineering", 120000), ("Bob", "Engineering", 110000), ("Charlie", "Engineering", 105000), ("David", "Sales", 90000), ("Eve", "Sales", 95000) ], ["name", "department", "salary"]) # הגדרת window window_spec = Window.partitionBy("department").orderBy(col("salary").desc()) # הוספת rank ranked = employees.withColumn("rank", rank().over(window_spec)) # Top 3 per department top_3 = ranked.filter(col("rank") <= 3) # משכורת ממוצעת per department window_spec2 = Window.partitionBy("department") with_avg = top_3.withColumn("dept_avg_salary", avg("salary").over(window_spec2)) with_avg.show()
תרגיל 5
Structured Streaming - ספירה בזמן אמת

משימה: ספור הודעות בזמן אמת מ-Kafka.

  • קרא stream מ-Kafka topic
  • המר את value ל-string
  • פצל למילים וספור
  • כתוב לconsole עם update mode
  • הוסף checkpoint
# פתרון from pyspark.sql.functions import explode, split # קריאה מ-Kafka df = spark.readStream \ .format("kafka") \ .option("kafka.bootstrap.servers", "localhost:9092") \ .option("subscribe", "messages") \ .load() # המרה ל-string lines = df.selectExpr("CAST(value AS STRING)") # פיצול וספירה words = lines.select(explode(split(col("value"), " ")).alias("word")) word_counts = words.groupBy("word").count() # כתיבה לconsole query = word_counts.writeStream \ .outputMode("update") \ .format("console") \ .option("checkpointLocation", "/tmp/checkpoint") \ .start() query.awaitTermination()

תרגיל 7
UDF - קטגוריזציה מותאמת אישית

משימה: צור UDF לקטגוריזציה של ציונים.

  • 90-100: Excellent
  • 80-89: Good
  • 70-79: Average
  • מתחת ל-70: Need Improvement
# פתרון from pyspark.sql.functions import udf from pyspark.sql.types import StringType # הגדרת UDF @udf(returnType=StringType()) def grade_category(score): if score >= 90: return "Excellent" elif score >= 80: return "Good" elif score >= 70: return "Average" else: return "Need Improvement" # נתוני דוגמה students = spark.createDataFrame([ ("Alice", 95), ("Bob", 85), ("Charlie", 75), ("David", 65) ], ["name", "score"]) # שימוש ב-UDF result = students.withColumn("grade", grade_category(col("score"))) result.show()

בונוס: המר ל-Pandas UDF לביצועים טובים יותר!

# Pandas UDF - יעיל יותר! from pyspark.sql.functions import pandas_udf import pandas as pd @pandas_udf("string") def grade_category_pandas(scores: pd.Series) -> pd.Series: def categorize(score): if score >= 90: return "Excellent" elif score >= 80: return "Good" elif score >= 70: return "Average" else: return "Need Improvement" return scores.apply(categorize) result = students.withColumn("grade", grade_category_pandas(col("score"))) result.show()
