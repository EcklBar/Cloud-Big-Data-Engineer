Hands-on Practice Lab

Test your knowledge with these scenarios. Try to solve them before looking at the solution.

SETUP Create Index & Bulk Index Data

Create an index named bookdb_index with 1 shard, and index 4 book documents using the _bulk API.

PUT /bookdb_index { "settings": { "number_of_shards": 1 }} POST /bookdb_index/book/_bulk { "index": { "_id": 1 }} { "title": "Elasticsearch: The Definitive Guide", "authors": ["clinton gormley", "zachary tong"], "summary" : "A distibuted real-time search and analytics engine", "publish_date" : "2015-02-07", "num_reviews": 20, "publisher": "oreilly" } { "index": { "_id": 2 }} { "title": "Taming Text: How to Find, Organize, and Manipulate It", "authors": ["grant ingersoll", "thomas morton", "drew farris"], "summary" : "organize text using approaches such as full-text search, proper name recognition, clustering, tagging, information extraction, and summarization", "publish_date" : "2013-01-24", "num_reviews": 12, "publisher": "manning" } { "index": { "_id": 3 }} { "title": "Elasticsearch in Action", "authors": ["radu gheorge", "matthew lee hinman", "roy russo"], "summary" : "build scalable search applications using Elasticsearch without having to do complex low-level programming or understand advanced data science algorithms", "publish_date" : "2015-12-03", "num_reviews": 18, "publisher": "manning" } { "index": { "_id": 4 }} { "title": "Solr in Action", "authors": ["trey grainger", "timothy potter"], "summary" : "Comprehensive guide to implementing a scalable search engine using Apache Solr", "publish_date" : "2014-04-05", "num_reviews": 23, "publisher": "manning" }
02 Return all documents
GET /bookdb_index/book/_search
03 Multi-match Search

Search for documents holding the word "guide" examining all fields.

GET /bookdb_index/book/_search { "query": { "multi_match" : { "query" : "guide" } } }
04 Specific Field Search

Search the phrase "action" in the title field.

POST /bookdb_index/book/_search { "query": { "match" : { "title" : "in action" } } }
05 Boosting Relevance

Add a boost of 3 when searching the phrase "search analytics" in all documents.

POST /bookdb_index/book/_search { "query": { "multi_match" : { "query" : "search analytics", "boost" : 3 } } }
06 Boolean Logic

Find books with "Elasticsearch" OR "Solr" in the title, AND authored by "Clinton Gormley", but NOT "Radu Gheorge".

POST /bookdb_index/book/_search { "query": { "bool": { "must": { "bool" : { "should": [ { "match": { "title": "Elasticsearch" }}, { "match": { "title": "Solr" }} ], "must": { "match": { "authors": "clinton gormely" }} } }, "must_not": { "match": {"authors": "radu gheorge" }} } } }
07 Wildcard & Highlight

Find records where author name starts with 't' and highlight the author field.

POST /bookdb_index/book/_search { "query": { "wildcard" : { "authors" : "t*" } }, "highlight": { "fields" : { "authors" : {} } } }
08 Regexp Search

Find authors starting with 't' and ending with 'y'.

POST /bookdb_index/book/_search { "query": { "regexp" : { "authors" : "t[a-z]*y" } } }
09 Phrase Prefix

Find documents with the phrase "search en" in the summary field.

POST /bookdb_index/book/_search { "query": { "match_phrase_prefix" : { "summary": { "query": "search en" } } } }
10 Range Query

Search for books published in 2015 only.

POST /bookdb_index/book/_search { "query": { "range" : { "publish_date": { "gte": "2015-01-01", "lte": "2015-12-31" } } } }
11 Filtering Results

Books with "Elasticsearch" in title/summary AND 20 or more reviews (Filter context).

POST /bookdb_index/book/_search { "query": { "bool": { "must" : { "multi_match" : { "query": "elasticsearch", "fields": [ "title", "summary" ] } }, "filter": { "range" : { "num_reviews": {"gte": 20} } } } } }

Using Kibana, create a dashboard of your choice based on the index you just created. (e.g., Pie chart of Publishers, or Reviews over time).
