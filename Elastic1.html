📚 Elasticsearch REST API Reference

Elasticsearch uses standard HTTP verbs to interact with data. Below are the core commands you need to know.

GET Retrieve Data
Used to retrieve information about a document, an index, or the cluster status. It does not modify data.
# Get a specific document by ID
GET /my-index/_doc/1

# Search for all documents
GET /my-index/_search
POST Create / Complex Search
Used to create a new document (when you don't have an ID) or to send complex search queries (Request Body Search).
# Create doc (Auto-generated ID)
POST /my-index/_doc
{ "user": "kimchy", "message": "trying out Elasticsearch" }

# Complex Search Query
POST /my-index/_search
{ "query": { "match": { "message": "trying out" } } }
PUT Create (Specific ID) / Update
Used to create a document when you know the ID, or to create a new Index.
# Create/Update specific doc ID 1
PUT /my-index/_doc/1
{ "user": "kimchy", "post_date": "2023-01-01" }

# Create a new Index
PUT /new-index-2023
DELETE Remove Data
Used to remove a specific document or delete an entire index (be careful!).
# Delete a document
DELETE /my-index/_doc/1

# Delete an entire index
DELETE /my-index

🎓 Core Concepts

  • Beats: Lightweight agents that sit on your servers and ship data (logs, metrics) to the stack.
  • Logstash: The processing pipeline. It takes raw data, parses it (Grok), filters it, and sends it to storage.
  • Elasticsearch: The search engine. It stores data in JSON format inside Indices. Indices are split into Shards for scalability.
  • Kibana: The UI. It lets you query Elasticsearch and build dashboards.
