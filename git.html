מה זה Version Control? [cite: 1551]

יישום שמאחסן ומנהל כל גרסה (Revision) של הקבצים והקוד שלך.

  • שיתוף פעולה על קוד מקור וניהול גרסאות[cite: 1553].
  • חזרה אחורה (Roll back) לגרסאות קודמות במקרה של באגים[cite: 1554].

למה Git? [cite: 1559]

  • נוצר על ידי Linus Torvalds ב-2005 (עבור ה-Linux Kernel)[cite: 1560].
  • מבוזר (Distributed): לכולם יש עותק מלא של הקוד וההיסטוריה[cite: 1595].
  • מחליף כלים ישנים כמו CVS, Subversion[cite: 1561].
  • מבטיח את שלמות הקבצים (File Integrity)[cite: 1564].

מושגי יסוד ומבנה הנתונים [cite: 1631]

אחסון דלתא (Delta Storage) [cite: 1610]

Git משתמש במנגנון חכם לאחסון שינויים:

  • במקום לשמור עותקים מלאים, Git שומר את השינויים (Deltas) בין הגרסאות[cite: 1612].
  • כל Commit הוא "תמונת מצב" (Snapshot) של הפרויקט[cite: 1613].
  • זה מאפשר חיסכון משמעותי במקום וניהול היסטוריה יעיל[cite: 1614].

אובייקטים ב-Git [cite: 1705]

  • Blob: התוכן הגולמי של הקובץ (בינארי)[cite: 1706].
  • Tree: מייצג תיקייה (שמות קבצים, הפניות ל-Blobs, תתי-תיקיות)[cite: 1707].
  • Commit: גרסה של ה-Tree + מטא-דאטה (מחבר, תאריך)[cite: 1711].
  • Tag: תווית לקומיט ספציפי (למשל גרסה V1.0)[cite: 1714].

מצבי קבצים [cite: 1764]

  • Untracked: קובץ חדש ש-Git עדיין לא עוקב אחריו.
  • Modified: קובץ קיים ששונה אך טרם הוכן לשמירה.
  • Staged: קובץ עם שינויים שמוכן ל-Commit הבא.
  • Committed: השינויים נשמרו במאגר המקומי.

פקודות בסיסיות [cite: 1797]

git init # אתחול מאגר חדש git clone <url> # העתקת מאגר קיים git status # בדיקת מצב הקבצים git add <file> # העברה ל-Staging git commit -m "msg" # שמירת גרסה git push # שליחה לשרת

ענפים (Branching) [cite: 1820]

התייחס לקוד כעץ. כל ענף הוא סט ליניארי של Commits. זה מאפשר עבודה במקביל ללא הפרעה לקוד הראשי.

# יצירת ענף חדש ומעבר אליו [cite: 1833] git checkout -b feature_branch # הצגת כל הענפים [cite: 1835] git branch -a # חזרה לענף הראשי [cite: 1842] git checkout main # מיזוג ענף צדדי לראשי [cite: 1843] git merge feature_branch

תרחישי מיזוג

  • Fast-Forward: אין שינויים במאסטר, המצביע פשוט זז קדימה.
  • Merge Commit: היסטוריה מתפצלת, נוצר Commit מיזוג חדש.
  • Conflict: שינויים באותה שורה בשני הענפים[cite: 1848].

ניהול התנגשויות (Conflicts) [cite: 1856]

כאשר שני אנשים משנים את אותו הקובץ, Git עוצר ומבקש עזרה.

תרגילים מעשיים [cite: 1948]

📝 תרגיל 1: בסיס [cite: 1952]
  1. אתחל מאגר חדש (git init).
  2. צור קובץ בשם name.txt עם השם שלך.
  3. בצע Stage לקובץ (git add).
  4. צור Commit חדש (git commit).
🌿 תרגיל 2: ענפים [cite: 1959]
  1. צור ענף חדש בשם side_branch.
  2. שנה את התוכן של name.txt לשם של חבר.
  3. בצע Stage ו-Commit לשינויים בענף החדש.
  4. צפה בהיסטוריה באמצעות git log ו-git status.
🔀 תרגיל 3: מיזוג וקונפליקטים [cite: 1970]
  1. חזור לענף הראשי (main/master).
  2. נסה למזג את side_branch לתוך הראשי.
  3. פתור את הקונפליקט שנוצר כך ששני השמות יופיעו.
  4. השלם את המיזוג וצפה בעץ ה-Git המלא.
