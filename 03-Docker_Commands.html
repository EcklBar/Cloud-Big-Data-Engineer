üìö Docker Definitions

Docker Image
A read-only template containing all the necessary files, libraries, and dependencies required to run an application in a Docker container.
Docker Container
A running instance of an image. That is, an executable package including the code, libraries, and dependencies needed to run the application.
Dockerfile
A script containing instructions to create the image.
Docker Registry
A repository for storing, sharing, and managing Docker images. These include Docker Hub, Amazon Elastic Container Registry, Microsoft Azure Container Registry, and Google Cloud Container Registry.
Docker Engine
An application for managing Docker containers. It includes a server (the "Docker daemon"), a command line tool (the Docker client), and an API for other software to interact with the Docker Daemon.
Docker Client
A command-line tool to interact with Docker Engine to manage Docker images and containers.
Docker Daemon
A background process that manages Docker images and containers according to the commands sent from the Docker client.
Docker Compose
A tool for defining and running multi-container Docker applications using a YAML file to configure application services.

üöÄ Running Containers

Run a container
docker run [image]
Run a test container to check your installation works
docker run hello-world
Run a container, then use it to run a command
docker run [image] [command]
Run Python interactively
docker run python -c "print('Python is Docker')"
Run R and print a model
docker run rocker/r-base r -e "print(lm(dist~speed, cars))"
Run a container interactively
docker run --interactive --tty rocker/r-base
Run a container, and remove it once you've finished
docker run --rm my_mysql
Run an image in the background
docker run --detach postgres
Run an image, assigning a name
docker run --name [name] redis
Run an image as a user
docker run --user doctordocker mongo
